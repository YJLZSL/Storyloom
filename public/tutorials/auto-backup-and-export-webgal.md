# 自动备份与导出 WebGAL

Storyloom（织叙）为长篇创作提供**自动备份**保护机制，确保你的故事数据不会因意外而丢失。同时，Storyloom 支持将作品导出为 **WebGAL 格式**，让你创作的视觉小说可以直接在浏览器中运行。

## 功能概述

### 自动备份系统

Storyloom 的自动备份功能在**本地**持续保护你的创作数据：

- **定时备份**：按设定的时间间隔（如每 5 分钟、每 10 分钟）自动创建备份快照
- **增量备份**：仅保存自上次备份以来的变更，节省存储空间
- **版本历史**：保留多个历史版本，可以回溯到任意时间点的状态
- **本地存储**：所有备份数据保存在你的本地磁盘，不上传云端，保障隐私安全
- **崩溃恢复**：如果 Storyloom 意外关闭，下次启动时会检测并提示恢复未保存的数据

### 导出 WebGAL

WebGAL 是一种基于 Web 的视觉小说引擎格式，Storyloom 可以将你的故事直接导出为可运行的 WebGAL 项目：

- 章节结构自动转换为 WebGAL 的剧本脚本
- 角色信息映射为 WebGAL 的角色定义
- 对话和旁白格式化为 WebGAL 的文本命令
- 支持导出为独立的 HTML 文件或完整的项目文件夹
- 导出的项目可以直接用浏览器打开，无需服务器部署

## 操作指南

### 查看备份

1. 点击顶部菜单「工具」→「备份管理」，或在命令面板搜索「备份」
2. 打开「备份管理器」面板，你会看到：
   - **备份列表**：按时间倒序排列的所有备份快照
   - **备份时间**：精确到分钟/秒的创建时间
   - **变更摘要**：显示该备份相比上一个版本的主要变更（如「新增角色：林雪」「修改第三章内容」）
   - **备份大小**：每个快照占用的磁盘空间
3. 点击某个备份条目，可以预览该版本的故事结构快照

> **提示**：备份管理器会用不同颜色标记备份类型：
> - 🟢 绿色：自动定时备份
> - 🟡 黄色：手动触发的备份
> - 🔴 红色：崩溃恢复点

### 恢复备份

当你需要回滚到某个历史版本时：

**方式一：完全恢复**

1. 在备份管理器中，找到想恢复的时间点
2. 点击该条目右侧的「恢复」按钮
3. 系统会弹出确认对话框，提示「恢复将覆盖当前数据，是否继续？」
4. 确认后，Storyloom 会自动：
   - 先创建当前状态的**临时备份**（防止误操作）
   - 将选中备份版本的内容完整恢复到工作区
   - 刷新界面，显示恢复后的内容

**方式二：部分恢复**

1. 在备份管理器中点击某个备份条目，进入**预览模式**
2. 在预览界面中，你可以：
   - 浏览该备份版本的所有章节、角色、世界观文档
   - 选择某个具体文件（如「第三章.md」或「主角设定.json」）
   - 点击「仅恢复此文件」
3. 这样只会恢复选中的文件，不会覆盖其他未改动的内容

> **重要提示**：恢复操作不可逆（虽然临时备份提供了「撤销恢复」的途径）。建议在恢复前先手动备份当前工作，或确保你确实需要回滚。

**撤销恢复**：

如果你恢复后发现问题，可以在备份管理器顶部点击「撤销上次恢复」，系统会回退到恢复前的状态。

### 设置自动备份

1. 进入「设置」→「数据与备份」
2. 在「自动备份」区域配置：
   - **备份间隔**：可选「每 2 分钟」「每 5 分钟」「每 10 分钟」「每 30 分钟」
   - **最大保留数量**：设置最多保留多少个备份快照（如 50 个），超过后会自动删除最旧的
   - **备份位置**：默认保存在 Storyloom 数据目录下，也可指定自定义文件夹
   - **包含媒体文件**：是否将图片、音频等素材也一并备份（占用空间更大，但恢复更完整）
3. 点击「保存」

> **存储空间建议**：如果每 5 分钟备份一次、保留 50 个版本，且故事文本量较大，建议预留至少 500MB 的备份空间。如果开启「包含媒体文件」，则根据素材总量预留更多空间。

### 导出 WebGAL

**准备工作**：

WebGAL 格式对内容结构有一定要求。导出前建议：
- 确保角色有对应的立绘图片（放在 `assets/characters/` 目录下，或后续手动添加）
- 确保场景有背景图（放在 `assets/backgrounds/` 目录下）
- 检查对话格式，确保发言者名称与角色名一致

**导出步骤**：

1. 点击顶部菜单「文件」→「导出」→「导出为 WebGAL」
2. 在导出向导中配置：
   - **导出范围**：
     - 「全部」：导出整个故事的所有章节
     - 「选择章节」：只勾选你想导出的章节（适合分卷发布）
   - **输出格式**：
     - 「完整项目文件夹」：包含 HTML、脚本、资源的完整文件夹，可二次编辑
     - 「单文件 HTML」：所有内容打包为一个 `.html` 文件，可直接双击打开
   - **立绘映射**：为每个角色指定默认立绘文件名（如果已提前放入素材目录）
   - **标题与封面**：设置 WebGAL 运行时的标题和封面图
3. 点击「导出」，选择保存位置
4. 等待导出完成，系统会提示导出成功并打开目标文件夹

**导出后的结构**：

```
MyStory_WebGAL/
├── index.html          # 入口文件
├── assets/
│   ├── characters/     # 角色立绘
│   ├── backgrounds/    # 场景背景
│   └── audio/          # 音乐与音效
├── script/             # 剧本脚本
│   ├── main.txt        # 主剧本（由章节自动合并）
│   └── start.txt       # 启动脚本
└── config.json         # WebGAL 配置
```

**运行导出的 WebGAL**：

- **方式一**：直接双击 `index.html` 文件，用浏览器打开
- **方式二**：将项目文件夹部署到任何静态网站托管服务（如 GitHub Pages、Vercel、Netlify）
- **方式三**：使用 WebGAL 官方提供的打包工具生成桌面端可执行文件

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + Shift + B` | 打开备份管理器 |
| `Ctrl + Shift + E` | 打开导出向导 |
| `Ctrl + Shift + S` | 立即手动创建备份快照 |

## 最佳实践

1. **设置合理的备份频率**：如果你正在高强度创作（如赶稿期），建议设为每 2 分钟备份一次；如果处于构思阶段改动较少，每 10 或 30 分钟即可。频率越高，占用空间越多。

2. **在重大修改前手动备份**：在执行「批量替换角色名」「合并章节」「结构调整」等高风险操作前，按 `Ctrl + Shift + S` 手动创建一个备份点。这样即使操作失误，也能精确回滚。

3. **定期清理旧备份**：虽然 Storyloom 会自动删除超出「最大保留数量」的旧备份，但建议每月进入备份管理器查看一次，手动删除明显不需要的早期快照（如「新建文档」阶段的空白备份）。

4. **WebGAL 导出前做好素材规划**：WebGAL 的视觉呈现需要立绘和背景图。建议在 Storyloom 中写作时就为角色准备好立绘文件名，并在导出配置中正确映射，这样导出后无需大量修改脚本。

5. **分卷导出**：对于超长篇作品，不要一次性导出全部章节。利用「选择章节」功能，按卷或按章节范围分批导出，便于分阶段测试和发布。

6. **备份位置选择独立磁盘**：如果条件允许，将备份目录设置为与 Storyloom 数据目录不同的物理磁盘（如外置硬盘或 NAS）。这样即使主硬盘故障，备份数据仍然安全。
