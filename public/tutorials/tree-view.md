# 树状视图教程

树状视图是 Storyloom（织叙）中专门用于可视化与管理分支叙事的独特工具。当故事出现多结局、角色抉择、时间线分叉或「如果……会怎样」的平行展开时，树状视图将这些可能性以一棵自上而下的决策树清晰呈现，让你一目了然地掌控所有剧情分支的走向与覆盖状态。

## 功能概述

树状视图提供以下核心能力：

- **分支可视化**：将故事的每个关键抉择点渲染为树节点，不同选择延伸出不同分支，形成完整的剧情树。
- **多结局检查**：通过颜色或标记快速识别哪些分支已通向完结、哪些仍在悬而未决，确保没有遗漏的「断尾」分支。
- **节点折叠/展开**：面对庞大的分支树时，可折叠已完成或暂不关注的路径，保持工作区整洁。
- **分支状态追踪**：每个节点可标记「已编写」「待扩展」「废弃」「草稿」等状态，直观反映分支内容的成熟度。
- **与主线关联**：分支节点可与时间轴视图中的事件双向关联，保证「主线剧情 → 分支触发 → 分支展开 → 回归主线」的闭环完整性。
- **深度导航**：支持聚焦到单条分支路径，暂时屏蔽其他分支，沉浸式编辑一条特定故事线。

## 操作指南

### 查看分支树

1. 点击顶部工具栏的 **「树状视图」** 按钮，或按下 `Ctrl + 3` 切换至该视图。
2. 主区域将展示以当前故事的「起始节点」为根的一棵剧情树。根节点通常对应故事的起点或第一个关键抉择点。
3. 从左到右阅读：每个节点显示标题和状态色标；从节点向下延伸的连线代表「选择」或「事件结果」，通向子节点。
4. 若分支较多，可使用右下角的 **迷你地图** 快速定位，或按 `H` 将整棵树适配到视口中央。

> 💡 节点颜色含义：绿色=已完成，黄色=待扩展/草稿，灰色=废弃，红色=存在逻辑冲突待修复。将鼠标悬停在节点上可查看详细说明。

### 添加分支节点

1. 选中一个现有节点（作为父节点），该节点代表触发抉择的「事件」或「状态」。
2. 点击节点右侧的 **「+ 添加分支」** 按钮，或右键选择 **「添加子节点」**。
3. 在弹出的编辑器中填写：
   - **分支标题**：描述该分支的核心走向，如「选择相信陌生人」或「真相线：幕后黑手是导师」。
   - **触发条件**：什么条件下这条分支被激活（可以是玩家选择、随机事件、前置剧情等）。
   - **内容摘要**：该分支大致会发生哪些关键事件、涉及哪些角色、预期情绪走向。
   - **是否完结**：勾选则表示该分支通向故事结局（如 Bad End 或 True End）。
   - **关联节点**：绑定到大纲视图中的对应场景或时间轴中的事件，实现跨视图联动。
4. 点击 **保存**，新分支将以子节点形式出现在父节点下方，与兄弟节点并列。

> 💡 建议每个抉择节点提供 **2-4 个分支选项**。超过 4 个会让树快速膨胀，管理成本陡增；若确有更多可能性，可在下级再设次级抉择。

### 折叠与展开

- **单击节点前的箭头图标**：切换该节点下所有子分支的显示/隐藏。折叠后，该节点右下角会显示一个数字徽章，提示其下隐藏的子节点数量。
- **右键菜单批量操作**：右键任意节点，选择 **「折叠所有子节点」** 可一键收起该分支下的全部层级；选择 **「展开所有子节点」** 可逐层展开至最深处。
- **按状态折叠**：使用顶部工具栏的 **「折叠已完成」** 按钮，一次性将所有标记为「已编写」或「废弃」的叶子节点折叠，只保留待创作和草稿节点，形成「待办清单」视角。
- **路径聚焦**：右键节点选择 **「聚焦此路径」**，视图将只显示从根节点到该节点的直系路径，隐藏所有旁支，适合沉浸式编写单线剧情。点击顶部面包屑的 **「退出聚焦」** 恢复全貌。

### 连接与合并分支

- **分支回归**：两条原本独立的支线可能在后续汇合。操作方式：在目标汇合节点上右键选择 **「合并来源」**，然后从树上选择需要汇入的另一条支线的尾节点。合并后，两条路径将共同指向后续节点，树结构变为有向无环图（DAG）形态。
- **标记冲突**：若两条分支的内容存在逻辑矛盾（如 A 分支中角色已死亡，B 分支中该角色仍存活），在冲突节点上右键标记 **「存在逻辑冲突」**，系统会在该节点显示红色警告角标，提醒你后续修复。
- **废弃分支**：对于已放弃的创作方向，不要直接删除，而是将其标记为「废弃」。这样既保留了创作历史，又避免树状图继续膨胀，废弃节点在默认视图下会以半透明显示。

### 跨视图导航

1. 在树状节点上点击 **「在时间轴中查看」**，将自动切换至时间轴视图并定位到该分支对应的轨道事件。
2. 在树状节点上点击 **「在大纲中查看」**，将跳转至大纲视图中绑定的大纲节点。
3. 这种双向联动确保了「结构规划（大纲）→ 时间编排（时间轴）→ 分支管理（树状）」三位一体的创作体验。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + 3` | 切换到树状视图 |
| `+` / `-` | 放大 / 缩小树状图 |
| `滚轮` | 垂直滚动浏览树状图 |
| `Ctrl + 滚轮` | 缩放树状图 |
| `空格 + 拖拽` | 平移整个画布 |
| `Enter` | 为选中节点添加子分支 |
| `Delete` | 删除选中节点（废弃节点可被彻底删除） |
| `空格` | 展开/折叠选中节点的子节点 |
| `F` | 聚焦选中节点（居中并放大） |
| `H` | 适配整棵树到视口 |
| `Ctrl + 单击` | 多选节点（用于批量标记状态） |
| `Esc` | 退出聚焦模式 / 取消多选 |

## 最佳实践

- **尽早规划「主干」**：在分支树生长前，先确定一条「主线」（最可能或最标准的剧情走向），将其作为树的主干。其他分支作为从主干延伸的旁枝， mentally model 更清晰，避免全员皆支线的混沌。
- **定期「修剪」**：每完成一轮创作，检查分支树中是否存在「死胡同」——即没有子节点、没有标记完结、也长时间未编辑的悬空节点。这些往往是创作过程中遗忘的伏笔，需要决定是补完还是废弃。
- **用颜色做「健康检查」**：在缩略视角下观察整棵树的颜色分布。如果大部分节点都是黄色（待扩展），说明项目尚处于早期；如果绿色和灰色过多而黄色极少，说明项目接近完成。红色角标则应零容忍，发现立即处理。
- **控制树的「宽度爆炸」**：分支叙事最常见的陷阱是每到一个抉择点就分裂，导致树呈指数级膨胀。建议每隔 3-5 个节点设置一个「分支回归点」，让分散的线索收束，既降低管理复杂度，也避免读者/玩家产生选择疲劳。
- **为完结分支写「结语录」**：每个通向结局的叶子节点，除了标记「完结」，建议在摘要中写 1-2 句话总结这个结局的调性与核心感受（如「 bittersweet — 主角赢了，但失去了挚友」）。这在你有多个结局时，是防止「同质化结局」问题的有效自检手段。
- **导出树图用于讨论**：在团队协作或向编辑/制作人汇报时，可将当前树状图导出为 PNG/SVG 图片。一张直观的剧情树比万字文档更有效地传递叙事结构思路。
