# 时间轴视图教程

时间轴视图是 Storyloom（织叙）中用于管理叙事时间线的核心工作区。它以横向轨道的方式将故事事件按时间顺序排列，帮助你直观地把握故事节奏、事件密度与因果关联。

## 功能概述

时间轴视图提供以下核心能力：

- **多轨道管理**：支持创建多条轨道（如主线、支线、回忆线、时间循环线等），每条轨道可独立管理事件，避免不同叙事线混杂。
- **事件卡片**：每个事件以卡片形式呈现，显示标题、时间点、类型标签等关键信息，一目了然。
- **缩放与平移**：使用 `Ctrl + 滚轮` 快速缩放时间比例，从宏观的整部小说跨度聚焦到单个小时内的密集事件；使用 `滚轮` 或拖拽空白区域平移浏览时间线。
- **轨道显隐控制**：可随时隐藏或显示特定轨道，聚焦当前需要编辑的叙事线，减少视觉干扰。
- **事件连接**：支持在时间轴上绘制事件之间的因果、触发、回忆等关联线，帮助构建复杂的叙事网络。

## 操作指南

### 创建轨道

1. 在时间轴视图左侧的轨道列表区域，点击右上角的 **「+ 新建轨道」** 按钮。
2. 在弹出的对话框中输入轨道名称，例如「主线·第三卷」或「暗线·王室阴谋」。
3. 选择轨道颜色（建议不同轨道使用不同颜色以便区分）。
4. 点击 **确认**，新轨道将出现在列表中，并在时间轴主区域展开为一条空白轨道。

> 💡 建议为每条轨道设置清晰的命名规则，例如按叙事层级（主线/支线）、时间属性（现在/过去/未来）或视角角色命名。

### 添加事件

1. 在目标轨道上的任意时间位置 **双击空白区域**，或右键选择 **「新建事件」**。
2. 在弹出的事件编辑器中填写：
   - **事件标题**：简要概括该事件的核心内容，如「主角发现密信」。
   - **时间定位**：设定事件发生的精确时间点（可精确到小时/分钟）。
   - **事件类型**：选择标签（如「战斗」「对话」「转折」「伏笔」），便于后续筛选。
   - **详细描述**：补充事件的场景描写、对话摘要、情绪基调等创作笔记。
   - **关联角色**：从角色库中勾选该事件涉及的人物，方便后续按角色追踪故事线。
3. 点击 **保存**，事件卡片将出现在时间轴对应位置。

### 编辑事件

1. **单击**时间轴上的事件卡片，右侧将弹出事件详情面板。
2. 在详情面板中修改任意字段，修改内容会 **实时自动保存**。
3. 如需修改事件在时间轴上的位置，可直接 **拖拽卡片** 到新的时间点；拖拽时时间轴会显示辅助刻度线，帮助精准定位。

### 拖拽调整时间

- **左右拖拽**：改变事件的时间点。拖拽时按住 `Shift` 可吸附到最近的小时刻度，按住 `Alt` 可精确到分钟级自由定位。
- **跨轨道拖拽**：将事件从一个轨道拖到另一个轨道，可实现事件在叙事线之间的转移（如从「回忆线」拖回「主线」表示回忆结束）。
- **批量调整**：框选多个事件卡片（按住 `Ctrl` 逐选或框选），整体拖拽可同时平移一组事件的时间，保持相对间隔不变。

### 连接事件

1. 选中一个事件卡片，点击卡片上的 **连接锚点**（卡片左右两侧的小圆点）。
2. 拖拽连线到另一个事件卡片上释放，建立连接。
3. 在弹出的连接类型菜单中选择关系类型：
   - **因果**：前事件导致后事件发生（最常用）。
   - **触发**：前事件是后事件的直接触发条件。
   - **回忆**：后事件是对前事件的回忆/倒叙。
   - **平行**：两条线上同时发生的事件。
   - **自定义**：输入你自己的关系描述。
4. 连接建立后，时间轴上会显示带有箭头的连线；点击连线可编辑或删除关系。

> 💡 大量连接会让时间轴变得复杂。建议仅在关键剧情节点之间建立连接，避免所有事件都相连。

### 轨道显示控制

- 在左侧轨道列表中，点击每条轨道前的 **眼睛图标** 可切换该轨道的显示/隐藏。
- 隐藏轨道中的事件仍然存在于项目中，只是暂时不在视图中显示，便于聚焦当前编辑的叙事线。
- 被隐藏轨道的事件在搜索和导出时依然可见。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + 滚轮` | 缩放时间轴比例（放大/缩小） |
| `滚轮` | 水平平移时间轴（左右滚动） |
| `按住空格 + 拖拽` | 拖拽平移整个画布 |
| `Ctrl + N` | 在当前轨道新建事件 |
| `Ctrl + T` | 新建轨道 |
| `Delete` | 删除选中事件（需二次确认） |
| `Ctrl + Z` / `Ctrl + Y` | 撤销 / 重做 |
| `Ctrl + 单击` | 多选事件 |
| `双击` | 编辑选中事件 |
| `H` | 一键适配全部内容到视口（Fit to View） |

## 最佳实践

- **轨道分层策略**：推荐设置「主线」「支线」「背景/设定」「时间标记」四层轨道。前两者承载故事内容，「背景/设定」轨道可放置世界观揭示事件（如「魔法体系首次解释」），「时间标记」轨道专门放置日期/节日标记，作为整线参考坐标。
- **时间密度控制**：避免在很短的时间跨度内堆积过多事件，否则时间轴会被过度压缩。若某几小时发生了大量事件，可考虑单独建立一个「小时级」子轨道进行放大管理。
- **颜色语义化**：为事件类型统一颜色，例如红色=战斗、蓝色=对话、紫色=伏笔。这样即使不阅读标题，也能通过颜色快速识别时间轴上的节奏分布。
- **定期整理连线**：每完成一个大章节，检查事件连接是否完整。遗漏的因果连接往往是后续剧情崩坏的前兆。
- **利用缩放做「节拍检查」**：将时间轴缩放到极限压缩视图，观察事件分布是否均匀；密集区可能意味着节奏过快，空白区则可能过于拖沓。
