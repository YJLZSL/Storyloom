# 大纲视图教程

大纲视图是 Storyloom（织叙）中组织长篇叙事结构的枢纽。它以层级列表的形式将整部作品的章节、场景、幕乃至更细粒度单元进行树状编排，同时支持细纲演进历史追踪，让创作从粗到精的每一步都可回溯、可对比。

## 功能概述

大纲视图提供以下核心能力：

- **层级化结构**：支持从「卷/部 → 章 → 节 → 场景 → 节拍」多级嵌套，构建清晰的叙事金字塔。
- **细纲演进历史**：每个节点都有独立的修订历史记录，可随时回溯到之前的大纲版本，查看创作思路的变化轨迹。
- **Diff 对比**：支持将当前大纲与任意历史版本进行差异对比，精确定位哪些内容被增删改。
- **拖拽排序**：支持同级节点之间的自由拖拽排序，以及跨层级的升降级操作（如将一个场景提升为独立章节）。
- **富文本摘要**：每个节点均可编写内容摘要，支持字数统计、标签标记、完成状态标记。
- **与正文联动**：大纲节点可与实际正文内容绑定，实现「大纲 ↔ 正文」双向导航。

## 操作指南

### 创建大纲节点

1. 在大纲视图左侧的结构树中，选中你希望添加子节点的父节点（例如选中「第三章」以添加其下的场景）。
2. 点击工具栏上的 **「+ 添加子节点」**，或右键父节点选择 **「添加子节点」**。
3. 输入节点标题，例如「场景一：雨夜初遇」或「节拍：主角内心挣扎」。
4. 按下 `Enter` 确认创建，新节点将出现在父节点下方，并自动缩进显示层级关系。
5. 在右侧的摘要面板中，填写该节点的内容摘要、预计字数、目标情绪、涉及角色等创作元数据。

> 💡 善用「节拍」级节点记录极细粒度的创作意图（如「此处需要一句让读者心头一紧的台词」），后续写正文时可直接参考。

### 拖拽排序

- **同级重排**：鼠标按住节点前的 **拖拽手柄**（六圆点图标），上下拖动即可调整同级节点之间的顺序。释放时会出现一条蓝色指示线，标明目标位置。
- **跨级移动**：将节点 A 拖拽到节点 B 上并短暂停留，A 会成为 B 的子节点。这适用于将某个场景从第三章移到第四章等结构调整。
- **快捷键升降级**：选中节点后，按 `Tab` 键将其降级为上一节点的子节点；按 `Shift + Tab` 将其提升一级，脱离当前父节点成为同级。
- **批量移动**：`Ctrl + 单击` 多选多个节点，整体拖拽可一次性重组多个单元。

> ⚠️ 跨级移动时，若目标节点已有关联正文，系统会提示是否同步转移正文内容，请根据实际需求选择。

### 版本历史与细纲演进

1. 选中任意大纲节点，点击右侧摘要面板顶部的 **「历史版本」** 按钮（时钟图标）。
2. 面板将展开为该节点的修订时间线，按时间倒序列出所有历史版本，每个版本显示：
   - 修改时间（精确到秒）
   - 修改人（协作模式下）
   - 字数变化（+/- 标识）
   - 版本备注（如果保存时填写了）
3. 点击任意历史版本，右侧将预览该版本的摘要内容，方便你回顾当时的创作思路。
4. 若希望恢复该版本，点击 **「恢复此版本」**；当前内容将被存入新的历史节点，可再次回退，确保万无一失。

> 💡 建议在每次重大结构调整（如合并章节、删除支线）前，手动点击 **「保存快照」** 并填写备注，为历史留下清晰的里程碑。

### Diff 对比

1. 在「历史版本」面板中，勾选两个需要对比的版本（一个作为基准，一个作为对比）。
2. 点击 **「对比差异」** 按钮，系统将以高亮形式展示：
   - **绿色（+）**：对比版本中新增的内容
   - **红色（-）**：对比版本中删除的内容
   - **黄色（~）**：被修改的内容，旧版本在左、新版本在右并列显示
3. 对比结果可以按段落折叠/展开，也可以导出为 `.diff` 文件存档备查。

> 💡 Diff 功能不仅适用于单节点，在根节点（整部作品）层级使用，可以宏观对比两个时间点的整体大纲结构变化。

### 状态与标签管理

- **完成状态**：每个节点可标记为「未开始」「写作中」「已完成」「待修订」四种状态，以颜色圆点区分。在大纲树中，已完成节点会显示为灰色，方便一眼定位尚未完成的单元。
- **标签**：添加自定义标签（如「高能」「待查资料」「需要润色」），并在顶部过滤器中按标签快速筛选节点。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 在当前节点下方创建同级新节点 |
| `Tab` | 将当前节点降级（成为上一节点的子节点） |
| `Shift + Tab` | 将当前节点升级（脱离父节点提升一级） |
| `Delete` | 删除选中节点（含子节点，需二次确认） |
| `空格` | 展开/折叠当前节点的子节点 |
| `Ctrl + ↑ / ↓` | 在同级节点间快速上下移动选中项 |
| `Ctrl + Z` / `Ctrl + Y` | 撤销 / 重做 |
| `Ctrl + F` | 在大纲中搜索节点标题或摘要内容 |
| `F2` | 重命名当前节点标题 |

## 最佳实践

- **先粗后细的迭代法则**：初期只建立「卷 → 章」两层，完成粗纲后再逐步下沉到场景和节拍。避免过早陷入细节，导致结构僵化。
- **利用历史做「分支实验」**：当对某个章节有两种写法犹豫时，复制当前节点（连同历史），在副本上大刀阔斧修改。不满意随时回退，主节点不受影响。
- **大纲即「契约」**：多人协作时，将大纲节点的摘要视为对正文内容的契约。写正文前，确保大纲摘要已通过团队共识，减少后续返工。
- **定期「层级体检」**：每完成一个大章节，检查大纲层级是否仍然合理。过长（超过 5000 字）的「场景」节点应考虑拆分；过短（不足 500 字）的「章」节点应考虑合并或降级。
- **状态标记驱动写作**：每天打开项目时，先通过状态筛选找到标记为「未开始」或「写作中」的节点，集中精力攻克，避免在已完成的章节上反复游荡浪费时间。
- **导出外部备份**：每周将大纲导出为 Markdown 或 OPML 格式，在 Git 或云盘中进行版本控制，即使遇到极端数据问题也能迅速恢复叙事骨架。
