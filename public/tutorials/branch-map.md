# 分支地图教程

## 功能概述

分支地图是 Storyloom（织叙）专为多结局、多线叙事设计的可视化剧情管理工具。它通过树状与网状混合布局，将故事中的关键选择、条件判断、结局节点以图形化方式呈现，帮助创作者在复杂的多分支结构中保持清晰的全局视野。无论是 Galgame 式的角色个人线、悬疑作品的推理分歧点，还是开放世界式的非线性探索，分支地图都能直观展示每一条剧情路径的走向与汇聚关系。

在分支地图中，创作者可以创建条件变量来驱动分支逻辑（如"好感度 ≥ 50""是否取得关键证物"），并为每个分支节点设置触发条件。系统支持实时预览功能，允许创作者模拟不同选择组合下的剧情走向，快速验证各条线路是否能正确通向预设结局。分支地图与剧本编辑器、关系图深度联动，确保宏观分支结构与微观场景文本始终同步一致。

## 操作指南

### 创建分支节点

1. 进入"分支地图"面板后，点击画布空白处的"新建节点"按钮，或右键选择"添加节点"。
2. 选择节点类型：
   - **场景节点**：对应剧本编辑器中的一个具体场景，是分支地图的基础构成单元。双击可跳转至该场景进行编辑。
   - **选择节点**：代表玩家需要做出选择的剧情分歧点，通常包含 2–4 个选项分支。每个选项可设置显示文本和后续跳转目标。
   - **条件节点**：代表由系统变量自动判断的隐性分支，不显示给玩家，但会根据预设条件将剧情导向不同路径。适合用于好感度判定、隐藏事件触发等。
   - **结局节点**：标记一条剧情线的终点，可设置结局名称、结局类型（如 Good End / Bad End / Normal End / True End）以及解锁条件。
3. 节点创建后，可在右侧属性面板中编辑节点名称、填写备注说明，并为场景节点关联剧本编辑器中的对应场景文件。
4. 节点支持拖拽调整位置，系统会自动优化连线布局以减少交叉，保持地图的可读性。

### 设置条件变量

1. 在顶部工具栏点击"变量管理"，打开全局变量面板。这里列出了项目中所有已定义的条件变量，例如 `favor_rin`（凛的好感度）、`has_evidence_A`（是否持有证物 A）、`chapter3_cleared`（第三章是否通关）等。
2. 点击"新建变量"，定义变量名称、数据类型（数值型、布尔型、字符串型）和初始值。命名建议采用英文蛇形命名法，并附带中文注释，便于在表达式中引用。
3. 为条件节点或选择节点的选项设置触发条件：
   - 选中目标节点或选项，在右侧属性面板找到"条件设置"区域。
   - 点击"添加条件"，选择要判断的变量，设置比较运算符（`>`、`>=`、`==`、`!=`、`<`、`<=`）和阈值。
   - 支持多条件组合：使用"与"（AND）或"或"（OR）逻辑连接多个条件表达式。例如：`favor_rin >= 50 AND has_key == true`。
4. 在剧本编辑器中，可以通过演出指令或选择支改变变量值。分支地图会自动检测剧本中的变量变更语句，并同步显示在变量追踪面板中，方便创作者检查变量在何时、何处被修改。
5. 变量更改历史支持按时间线回溯，若发现某条分支因变量设置错误而无法到达，可通过历史记录快速定位问题所在场景。

### 预览不同结局路径

1. 点击顶部工具栏的"路径预览"按钮，进入剧情模拟模式。系统会弹出初始状态配置面板，允许你设置各个变量的起始值（默认使用初始值）。
2. 点击"开始模拟"，Storyloom 会高亮显示当前可达的剧情路径，从起始节点开始逐步推进。遇到选择节点时，界面会暂停并展示各选项，创作者可手动选择一条路径继续；遇到条件节点时，系统会根据当前变量值自动判断走向。
3. 在预览过程中，左侧会显示当前的"剧情路径栈"，记录从起点到当前位置经过的所有节点，方便理解自己是如何到达此处的。
4. 预览时若发现某条期望中的路径无法到达（例如选择了某个选项后跳到了意料之外的节点），可点击"暂停模拟"，直接在该节点处检查其条件设置、跳转目标或变量状态，无需退出预览模式。
5. 支持"快速到达结局"功能：在预览面板中选择任意结局节点，点击"测试能否到达"，系统会自动尝试寻找一条从起点到该结局的可行路径，并给出路径报告（如经过哪些选择节点、需要满足哪些变量条件）。若不可达，会分析阻断原因（如某条件永远无法满足）。
6. 预览完成后，可导出当前模拟路径为文本报告，用于团队内部走查或作为 QA 测试用例。

### 连接与整理节点

1. 选中一个节点，点击其底部的连接点并拖动到目标节点上，即可创建剧情流向连线。连线上会标注连接类型（默认跳转/条件成立/条件不成立）。
2. 选中连线后，可在右侧面板设置跳转延迟（如等待 BGM 播放完毕后再进入下一场景）和过渡效果。
3. 使用"对齐"和"等距分布"工具，可以快速将多个同级节点排列整齐，使分支地图在节点增多时依然保持清晰美观。
4. 支持为节点添加颜色标签：例如将所有 Good End 结局节点标记为绿色，Bad End 标记为红色，隐藏路线标记为紫色，便于一眼区分剧情性质。

## 快捷键

| 快捷键 | 功能说明 |
|--------|----------|
| `Ctrl + N` | 新建节点 |
| `Ctrl + Shift + N` | 新建选择节点（带默认两个选项） |
| `Ctrl + L` | 进入连线模式（从选中节点引出连线） |
| `Delete` | 删除选中的节点或连线（场景节点会提示确认） |
| `Ctrl + F` | 搜索节点名称 |
| `Ctrl + P` | 打开路径预览模式 |
| `Ctrl + V` | 打开变量管理面板 |
| `空格键` | 在预览模式中继续/下一步 |
| `Ctrl + 0` | 居中显示整个分支地图 |
| `Ctrl + 鼠标滚轮` | 缩放画布 |
| `按住中键拖动` | 平移画布 |
| `Ctrl + S` | 保存分支地图布局与变量状态 |
| `F9` | 快速测试当前选中结局是否可达 |

## 最佳实践

1. **先画骨架，后填血肉**：在创作前期，先用场景节点和选择节点搭建出故事的主要分支骨架（如第一章共通线 → 第二章分线 → 各角色结局），暂不深入细节。待整体结构验证无误后，再为每个场景节点关联具体剧本内容。

2. **控制分支深度**：分支层级过深会导致开发成本指数级增长。建议主线分支控制在 3–4 层以内，支线分歧可通过条件节点在单场景内解决，避免过早产生过多的独立场景文件。

3. **变量命名要自解释**：避免使用 `var1`、`flag2` 等模糊命名。推荐使用 `好感度_角色名`、`持有_道具名`、`事件_章节_标识` 等结构化命名，并养成在变量面板中写备注的习惯，否则项目后期极易忘记某个变量的用途。

4. **定期运行全结局可达性检查**：项目中期开始，建议每周使用"测试能否到达"功能遍历所有结局节点，确保没有因剧本修改或变量调整而被意外阻断的结局线路。及早发现死胡同，避免后期大规模返工。

5. **利用颜色标签进行线路分类**：为不同角色的个人线分配专属颜色（如红色代表女主角 A 线，蓝色代表女主角 B 线），在地图缩略图模式下可以迅速判断各线路的篇幅分配是否均衡，防止某条线路过于单薄或冗长。

6. **与剧本编辑器和关系图联动**：在分支地图中发现某个结局的情感张力不足时，切换到关系图查看该结局涉及的核心角色关系是否得到了充分刻画；若需要修改具体台词，则双击对应场景节点进入剧本编辑器。三个模块协同使用，才能实现从结构到文本的闭环优化。

7. **保留旧版本分支快照**：在重大结构调整（如砍掉一条支线、合并两个结局）前，导出当前分支地图为 PNG 或 JSON 快照存档。一旦新方案效果不佳，可以快速对照旧版本进行回滚或参考。
