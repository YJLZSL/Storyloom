# 剧本编辑器教程

## 功能概述

剧本编辑器是 Storyloom（织叙）中用于单一场景精细化创作的核心文本工具。它专注于台词撰写与演出指令编排，为视觉小说、交互叙事、剧本杀以及多分支故事提供高效的写作环境。编辑器采用简洁的分栏式布局，左侧为脚本结构导航，中间为编辑区，右侧为实时预览与属性面板，确保创作者能够沉浸在单一场景的打磨中，同时保持对整体结构的掌控。

剧本编辑器原生支持 WebGAL 格式导出，这意味着创作者可以直接将完成的内容输出为可运行的视觉小说项目文件，无需手动转换格式。无论是角色对话、旁白叙述、场景切换，还是背景图、BGM、立绘演出等视觉要素，编辑器都提供了直观的配置方式，让写作者无需编写复杂代码即可实现丰富的视听效果。

## 操作指南

### 编辑场景剧本

1. 在左侧场景列表中选中目标场景，或直接点击"新建场景"按钮创建空白场景。每个场景对应一个独立的剧本文件，建议按章节或地点命名以保持清晰。
2. 场景的基本信息（场景名称、发生地点、时间线、所属章节）在右侧面板中编辑。这些信息会自动同步到项目大纲，方便全局管理。
3. 在编辑区中，使用"行类型"按钮切换当前行的内容类型：
   - **对话**：带有角色名称和台词内容的行，适合角色之间的交流。
   - **旁白**：无角色标签的叙述文本，适合环境描写、心理描写或全局性说明。
   - **演出指令**：控制视觉和听觉呈现的特殊语法行，用于切换背景、播放音效、移动立绘等。
   - **注释**：仅在编辑时可见，不会输出到最终剧本，适合写给自己或团队成员的备忘。
4. 编辑器支持自动补全：输入角色名首字母或拼音时会提示已有角色，输入 `[` 或 `(` 等符号时会触发演出指令模板提示。
5. 使用顶部工具栏的"折叠/展开"功能，可以按角色或行类型折叠剧本内容，便于在长篇场景中快速定位。

### 添加角色台词

1. 点击编辑区左侧的"+ 对话"按钮，或按快捷键 `Ctrl + D`，即可插入一个新的对话行。
2. 在对话行的角色名栏中，可直接输入新角色名（系统会自动注册到角色库）或从下拉菜单中选择已有角色。角色名右侧可设置当前情绪标签，如"平静""愤怒""惊讶"，这些标签在导出后可映射到对应的立绘表情差分。
3. 在台词输入框中撰写内容。编辑器支持基础的文本格式：使用 `**文本**` 表示加粗（常用于强调），使用 `*文本*` 表示斜体（常用于心理活动），使用 `__文本__` 表示下划线。
4. 若需要插入选择支（让玩家决定下一句回应），点击"插入选择"按钮，系统会生成选择支模板。每个选择支可设置目标跳转场景以及该选项触发的条件变量（详见分支地图教程）。
5. 连续的多行同角色台词会自动合并显示，并可通过拖拽行首的排序图标调整台词顺序。

### 设置背景与 BGM

1. 在编辑区的任意位置插入一行"演出指令"行，类型选择"背景"（`bg`）。
2. 点击资源选择器，从项目资源库中选择一张背景图。支持 JPG、PNG 格式，建议使用 1920×1080 分辨率的图片以保证最佳显示效果。
3. 可设置背景切换效果："直接切换""淡入淡出""横向推移""遮罩溶解"等，并为效果指定持续时间（毫秒）。
4. 同理，插入类型为"音乐"（`bgm`）的演出指令，选择资源库中的音频文件（支持 MP3、OGG、WAV）。可设置淡入时间、循环播放选项以及音量大小（0–100%）。
5. 使用"效果"（`se`）指令可插入音效，如关门声、雨声、心跳声等，配合台词时间轴使用，增强临场感。
6. 右侧预览面板会实时反映背景和音乐的变更，方便创作者即时感受视听氛围是否匹配文本情绪。

### WebGAL 导出

1. 完成场景编写后，点击顶部菜单栏的"导出"按钮，选择"导出为 WebGAL 项目"。
2. 在导出配置对话框中，确认以下选项：
   - **导出范围**：全部场景、仅选中场景，或按章节批量导出。
   - **资源打包**：是否将背景、立绘、BGM 等素材一并复制到输出目录。建议勾选，以确保导出项目可直接运行。
   - **脚本格式**：WebGAL 标准格式（`.txt` 脚本文件）。系统会自动将编辑器中的结构化内容转换为 WebGAL 的语法标签。
   - **变量与分支**：如果项目中使用了条件变量和分支，导出时会自动转换为 WebGAL 的变量系统和场景跳转逻辑。
3. 点击"确认导出"后，Storyloom 会生成一个完整的 WebGAL 项目文件夹，包含 `index.html`、`config.txt`、脚本目录以及资源目录。
4. 创作者可直接用浏览器打开 `index.html` 进行预览，或将项目部署到 Web 服务器、打包为 Electron 桌面应用，或上传到 WebGAL 托管平台。
5. 若后续在 Storyloom 中修改了剧本内容，重新导出并覆盖原目录即可增量更新，无需重新配置 WebGAL 工程。

## 快捷键

| 快捷键 | 功能说明 |
|--------|----------|
| `Ctrl + N` | 新建场景 |
| `Ctrl + D` | 插入对话行 |
| `Ctrl + Shift + D` | 插入旁白行 |
| `Ctrl + Shift + E` | 插入演出指令行 |
| `Ctrl + /` | 将当前行切换为注释/取消注释 |
| `Ctrl + S` | 保存当前场景剧本 |
| `Ctrl + P` | 打开右侧实时预览（全屏模式） |
| `Ctrl + F` | 在当前场景中搜索文本 |
| `Ctrl + G` | 跳转到指定行号 |
| `Tab` / `Shift + Tab` | 增加/减少当前行的缩进层级 |
| `Ctrl + Z` / `Ctrl + Y` | 撤销 / 重做 |
| `F5` | 快速导出并预览（调用默认浏览器） |

## 最佳实践

1. **场景切分原则**：每个场景应围绕一个核心目标展开（如揭示信息、制造冲突、展示人物关系），避免单场景过长（建议控制在 100 行以内）。过长的场景应拆分为多个子场景，通过背景变化或时间跳跃进行过渡。

2. **先写文本，后配资源**：在创作初稿阶段，建议先专注于台词和叙事文本的打磨，使用占位符标记需要的背景/音乐（如 `[BG: 学校走廊]`），待文本定稿后再统一配置视觉资源，避免频繁切换打断写作心流。

3. **统一演出指令规范**：在团队协作中，提前约定演出指令的命名规范（如立绘位置用 `left`/`center`/`right`，表情用 `normal`/`angry`/`sad`），可减少沟通成本并降低导出错误率。

4. **善用注释做版本备忘**：在修改关键台词或删除支线内容时，不要直接删除，而是先将其转为注释行，并标注修改日期和原因。这样既能保留创作痕迹，又不会影响导出文件。

5. **定期导出测试**：由于 WebGAL 引擎对语法格式有一定要求，建议每完成 2–3 个场景就导出测试一次，及时发现格式转换中的兼容问题（如特殊符号转义、变量名冲突等），避免在项目末期集中爆发。

6. **配合关系图检查角色一致性**：在写长段对话时，定期切换到关系图查看当前角色的关联网络，确保台词中提到的背景信息、人际关系与全局设定保持一致，防止吃书。
