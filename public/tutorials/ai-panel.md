# AI 面板

AI 面板是 Storyloom（织叙）内置的智能助手工作台，它将多个主流 AI 提供商统一整合到叙事创作流程中，让你在写作过程中随时获得灵感、润色与反馈。

## 功能概述

Storyloom AI 面板支持接入 7 个主流 AI 提供商：

- **DeepSeek**：国产高性能大模型，擅长中文创作与逻辑推理
- **Kimi（Moonshot）**：超长上下文窗口，适合分析整部作品的设定与伏笔
- **GLM（智谱清言）**：中文理解与生成能力出色，适合对话润色
- **MiniMax**：角色扮演与情感对话表现优异
- **SiliconFlow**：灵活的模型分发平台，支持多种开源模型切换
- **OpenAI**：GPT 系列模型，多语言创作与代码辅助
- **自定义**：支持填写任意兼容 OpenAI API 格式的第三方服务地址

AI 面板支持**多轮对话**与**上下文记忆**，你提出的问题会携带历史记录，AI 能够理解前后关联。更关键的是，Storyloom 支持将**工作区数据作为上下文**传递给 AI——你可以直接让 AI 分析当前故事的角色关系、时间线或世界观设定。

## 操作指南

### 配置 AI 密钥

首次使用前，需要先配置对应提供商的 API 密钥：

1. 点击界面右上角「设置」图标，进入 **AI 设置** 页面
2. 在左侧列表中选择你想使用的提供商（如 DeepSeek、Kimi 等）
3. 在右侧输入框中粘贴你的 **API Key**
4. 部分提供商（如自定义）还需要填写 **Base URL**（API 接口地址）
5. 点击「保存」，密钥将加密存储在本地配置中

> **提示**：API Key 是敏感信息，Storyloom 不会将其上传到任何服务器，仅保存在本地。建议定期轮换密钥以保障安全。

### 开始对话

1. 点击侧边栏的 **「AI 助手」** 图标（或按快捷键 `Ctrl + Shift + A`）打开 AI 面板
2. 在面板底部输入框中输入你想问的问题，例如：
   - 「帮我润色这段对话，让女主角的语气更冷淡一些」
   - 「根据当前章节的情节，设计一个合理的反转」
   - 「分析这段文字的节奏问题」
3. 按下 **Enter** 发送消息，AI 会在面板中显示回复
4. 你可以继续追问，AI 会基于之前的对话上下文进行回答

### 使用工作区上下文

这是 Storyloom AI 面板最具特色的功能。你可以将当前故事中的内容作为上下文注入对话：

1. 打开 AI 面板后，点击输入框上方的 **「上下文」** 按钮
2. 选择你想附加的内容类型：
   - **当前章节**：将当前正在编辑的章节全文作为上下文
   - **角色设定**：将某个角色的详细设定卡作为上下文
   - **世界观文档**：将世界观/背景文档作为上下文
   - **时间线**：将故事时间线作为上下文
3. 选择后，点击「发送」

此时 AI 会基于你提供的**真实故事数据**进行分析和回答，而不是凭空猜测。例如：「请根据这段角色设定，为第三章设计一段符合他性格的独白」

### 切换模型

在对话过程中，你可以随时切换不同的 AI 模型：

1. 在 AI 面板顶部，点击当前显示的模型名称（如「DeepSeek-V3」）
2. 下拉菜单会展示所有已配置密钥的可用模型
3. 点击想切换的模型，下一条消息将使用新模型回复
4. 不同模型的对话记录会保留在各自的会话中，互不干扰

> **提示**：建议为不同类型的任务选择不同模型。例如：需要超长文本分析时选择 Kimi，需要创意写作时选择 DeepSeek 或 MiniMax，需要多语言润色时选择 GPT 系列。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + Shift + A` | 打开/关闭 AI 面板 |
| `Enter` | 发送消息 |
| `Shift + Enter` | 输入框中换行 |
| `Ctrl + L` | 清空当前对话 |
| `↑` / `↓` | 在输入框中浏览历史发送记录 |

## 最佳实践

1. **为不同任务创建不同的「对话线程」**：Storyloom 支持在 AI 面板中创建多个对话线程。建议为「大纲规划」「角色设计」「润色校对」等不同任务分别创建线程，避免上下文混淆。

2. **善用工作区上下文**：不要只把 AI 当作通用聊天工具，而是让它真正成为你的「故事助手」。在分析角色时附加角色卡，在讨论情节时附加当前章节，AI 的回答质量会显著提升。

3. **分步提问**：对于复杂需求，建议拆分成多个小步骤。例如：先让 AI 分析角色性格，再让它基于分析结果写对话，最后进行润色。比一次性要求「帮我写一段好对话」效果更可控。

4. **对比不同模型的输出**：利用一键切换模型功能，对同一段问题分别用 DeepSeek 和 Kimi 回答，取两者之长进行融合。

5. **定期清理对话历史**：当某个对话线程过长时，AI 可能会因上下文压缩而遗漏早期信息。建议每完成一个阶段任务后新建线程。
