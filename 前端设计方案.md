# 前端设计方案

> 基于 12 款时间轴工具、6 款创意写作工具、5 款标杆级应用的设计调研，结合本项目定位，制定的前端设计方案。
>
> 设计哲学：**"如纸般沉浸"** — 让创作者忘记工具的存在，沉浸在故事世界中。
>
> ⚠️ V7 重构后：设计系统已简化为 2 主题（素纸亮色 / 墨黑暗色）+ 专注模式，移除了旧的 12 主题 + 3 视觉模式（cinematic/balanced/minimal）。动画引擎改为 Framer Motion 11，组件库采用 shadcn/ui。

## 二、视觉设计体系

### 2.1 设计语言：「墨与纸」

核心隐喻：**墨水在纸上的书写**

- 墨 = 内容层：文字、事件、角色 — 沉稳、有质感（暗色主题「墨黑」）
- 纸 = 空间层：背景、容器 — 有纹理、有温度（亮色主题「素纸」）
- 光 = 交互层：选中、悬停、聚焦 — 明亮、有引导性

### 2.2 色彩系统（V7 重构后：2 主题）

## 五、动效设计体系

> ⚠️ V7 重构后动画引擎改为 **Framer Motion 11**，替代 anime.js v4。所有动效使用 Framer Motion 的 `motion` 组件和 `AnimatePresence` 实现。

### 5.1 动效层级

| 层级 | 用途 | 时长 | 缓动 | 示例 |
|------|------|------|------|------|
| L0 即时反馈 | 按钮点击、开关切换 | 0-100ms | easeOut | 按钮按下缩放 0.97 |
| L1 微交互 | 悬停、焦点、状态变化 | 100-300ms | spring | 卡片浮起、输入框聚焦 |
| L2 过渡动画 | 面板展开、视图切换 | 300-500ms | spring | 面板滑入、页面切换 |
| L3 场景转换 | 模态框、全屏切换 | 400-700ms | easeInOut | 对话框出现、主题切换 |
| L4 叙事动画 | 加载、首次渲染 | 500-1500ms | stagger + spring | 时间轴加载、事件交错出现 |

#### 主题切换（View Transitions API）

```tsx
// useThemeStore 中使用 View Transitions API 实现圆形扩散切换
const toggleTheme = async (event: React.MouseEvent) => {
  const x = event.clientX;
  const y = event.clientY;
  const endRadius = Math.hypot(
    Math.max(x, window.innerWidth - x),
    Math.max(y, window.innerHeight - y)
  );

  if (!document.startViewTransition) {
    setTheme(newTheme);
    return;
  }

  const transition = document.startViewTransition(() => {
    setTheme(newTheme);
  });

  await transition.ready;
  document.documentElement.animate(
    { clipPath: [`circle(0px at ${x}px ${y}px)`, `circle(${endRadius}px at ${x}px ${y}px)`] },
    { duration: 500, easing: 'cubic-bezier(0.4, 0, 0.2, 1)', pseudoElement: '::view-transition-new(root)' }
  );
};
```

### 5.3 专注模式动效

## 六、主题系统设计

### 6.1 2 主题定义（V7 重构后）

V7 重构后从 12 主题简化为 2 主题，每个主题是完整的故事氛围：

| 主题 | 氛围 | 背景 | 适用场景 |
|------|------|------|---------|
| **素纸亮色（light）** | 清爽干净、专注高效 | 纸张质感纯色 | 日间创作、长时间阅读 |
| **墨黑暗色（dark）** | 沉静深邃、沉浸专注 | 深色纯色 | 夜间创作、专注模式 |

**设计原则**：
- 2 主题覆盖所有使用场景，不再需要多主题切换
- 通过语义色（事件/伏笔/角色）在 2 主题间保持一致的视觉语言
- 专注模式（focusMode）作为补充，提供全屏写作环境

### 6.2 主题切换体验

使用 View Transitions API 实现圆形扩散切换（`useThemeStore`）：
- 点击主题切换按钮时，从点击位置开始圆形扩散
- 新主题的颜色从点击位置向外扩展
- 过渡期间，卡片内容保持可读（先换背景，再换前景）
- 总时长 500ms
- 不支持 View Transitions API 的浏览器降级为即时切换

### 6.3 CSS 变量格式

V7 重构后 CSS 变量使用 RGB 通道格式，配合 Tailwind v4 的 `rgb(var(--xxx))` 使用：

```css
[data-theme="light"] {
  --background: 250 247 242;
  --foreground: 26 22 18;
  --primary: 99 78 138;
  /* ... */
}

[data-theme="dark"] {
  --background: 18 18 24;
  --foreground: 240 237 232;
  --primary: 156 134 196;
  /* ... */
}
```

在 Tailwind 中使用：`bg-background`、`text-foreground`、`bg-primary` 等。

## 七、数据可视化设计

### 7.1 故事弧线图

- **形式**：SVG 曲线图，横轴=时间/章节，纵轴=情感强度
- **交互**：
  - 悬停显示具体情节描述
  - 点击跳转到对应事件
  - 多条弧线叠加对比不同角色/线索
- **动画**：弧线从左到右绘制动画（Framer Motion + stroke-dashoffset），2000ms
- **颜色**：不同角色/线索用不同颜色

### 7.2 角色关系图

V7 重构后由 `RelationshipView` 组件实现（已用新 store + d3）：

- **形式**：力导向图（D3.js force-directed graph）
- **节点**：角色头像 + 名字，大小=出场频率
- **连线**：关系类型用不同颜色和线型
  - 亲情：实线，暖色
  - 友情：实线，蓝色
  - 对立：虚线，红色
  - 暗恋：点线，紫色
- **交互**：
  - 拖拽节点调整布局
  - 悬停节点高亮其所有关联
  - 点击节点展开角色详情
  - 双指缩放
- **动画**：初始布局有力模拟动画，节点弹性就位

### 7.3 情节密度热力图

- **形式**：时间轴上的色块热力图
- **颜色**：冷色（稀疏）→ 暖色（密集）
- **交互**：悬停显示该时段的事件数量和列表
- **动画**：热力图从左到右渐显

### 7.4 伏笔生命周期图

- **形式**：Sankey 图或流程图
- **节点**：伏笔状态（种植→发展→回收/废弃）
- **连线**：伏笔流转路径，宽度=伏笔数量
- **交互**：悬停显示具体伏笔列表
- **动画**：流动粒子沿连线方向移动

## 十、技术实现建议

### 10.1 动画技术选型（V7 重构后）

| 场景 | 技术 | 理由 |
|------|------|------|
| 声明式组件动画 | **Framer Motion 11**（V7 已集成） | React 原生、声明式、弹簧物理、AnimatePresence |
| 主题切换过渡 | View Transitions API | 浏览器原生、圆形扩散效果 |
| 滚动驱动动画 | Framer Motion useScroll + useTransform | 时间轴缩放时的视差效果 |
| 力导向图 | D3.js force simulation | 角色关系图标准方案 |
| SVG 路径动画 | Framer Motion + stroke-dashoffset | 故事弧线绘制 |
| 手势/拖拽 | Framer Motion drag | 事件拖拽排序 |

> ⚠️ V7 重构后唯一动画引擎为 Framer Motion 11。旧组件中的 anime.js v4 引用待迁移完成后移除。

**Framer Motion 基本用法**：
```tsx
import { motion, AnimatePresence } from 'framer-motion';

// 基础动画
<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  exit={{ opacity: 0, y: -20 }}
  transition={{ duration: 0.3, ease: 'easeOut' }}
>
  内容
</motion.div>

// 弹簧动画
<motion.div
  animate={{ scale: 1 }}
  transition={{ type: 'spring', stiffness: 80, damping: 10 }}
>
  内容
</motion.div>

// 列表交错
const containerVariants = {
  hidden: { opacity: 0 },
  visible: { opacity: 1, transition: { staggerChildren: 0.05 } }
};
const itemVariants = {
  hidden: { opacity: 0, y: 20 },
  visible: { opacity: 1, y: 0 }
};
```

### 10.2 组件库策略（V7 重构后）

| 场景 | 组件库 | 理由 |
|------|--------|------|
| 基础 UI（按钮/输入/对话框/下拉/标签页等） | **shadcn/ui**（V7 已集成，new-york 风格） | 与 Tailwind 深度集成，可定制，源码可控 |
| 复杂组件（Timeline/Calendar/Tree/Steps） | TDesign React | 成熟的企业级组件，节省开发时间 |
| 图标 | lucide-react + @icon-park/react | lucide 做基础，IconPark 补充专业图标 |
| 数据可视化 | D3.js + 自定义 SVG | 灵活度最高 |

**shadcn/ui 使用方式**：
- 配置文件：`components.json`（new-york 风格）
- 组件位置：`src/components/ui/`
- 添加新组件：`npx shadcn@latest add <component>`
- 已集成 18 个基础组件：button、input、dialog、dropdown-menu、select、tabs、tooltip、popover、command、scroll-area、separator、label、switch、checkbox、slider、badge、card、sheet

### 10.3 关键架构决策

1. **时间轴画布**：自研 SVG/Canvas 混合渲染，不依赖第三方时间轴组件
2. **状态管理**：Zustand 4 域 store + React Query（V7 重构后），不引入新状态库
3. **路由**：单页面应用，视图切换不使用路由，通过 useTimelineStore.viewMode 控制
4. **CSS**：Tailwind CSS v4 + CSS Variables 2 主题系统（V7 重构后），不引入 CSS-in-JS
5. **主题系统**：2 主题（light/dark）+ 专注模式，通过 `[data-theme]` 切换，不再支持多主题/视觉模式
