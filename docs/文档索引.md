## 关键配置检查清单

每次发版前必须确认以下配置：

1. `package.json#version` — 版本号递增
2. `package.json#build.publish.owner` — 必须是 `"YJSL"`（⚠️ 曾错误改为 `liteli1987gmail`，已修正）
3. `package.json#build.publish.repo` — 必须是 `"Storyloom"`
4. `src/components/settings/UpdateTab.tsx#REPO_RELEASES_URL` — 必须指向 `https://github.com/YJSL/Storyloom/releases`
5. `docs/发版指南.md` — 所有 URL 必须指向 `YJSL/Storyloom`

---

## ⚠️ 自动更新重要提醒

**每次发版必须同时上传 `latest.yml`！**

`latest.yml` 是 Electron 自动更新机制（`electron-updater`）的元数据文件。如果 Release 中缺少此文件，老版本用户点击「检查更新」时会报错：

```
更新出错：Cannot find latest.yml in the latest release artifacts
```

**正确做法**：构建完成后，检查 `release/` 目录下是否生成了 `latest.yml`，确认内容中的 `version` 和 `url` 正确，然后连同 `.exe` 一起上传到 GitHub Release。
