## 二、视觉设计体系

### 2.1 设计语言：「墨与纸」

核心隐喻：**墨水在纸上的书写**

- 墨 = 内容层：文字、事件、角色 — 沉稳、有质感（暗色主题「墨黑」）
- 纸 = 空间层：背景、容器 — 有纹理、有温度（亮色主题「素纸」）
- 光 = 交互层：选中、悬停、聚焦 — 明亮、有引导性

### 2.2 色彩系统（重构后：2 主题）

重构后采用 2 主题系统，通过 `[data-theme="light|dark"]` 切换。CSS 变量使用 RGB 通道格式（如 `--background: 250 247 242`），配合 Tailwind 的 `rgb(var(--background))` 使用。

## 五、动效设计体系

> ⚠️ 重构后动画引擎改为 **Framer Motion 11**，替代 anime.js v4。所有动效使用 Framer Motion 的 `motion` 组件和 `AnimatePresence` 实现。

### 5.1 动效层级

| 层级 | 用途 | 时长 | 缓动 | 示例 |
|------|------|------|------|------|
| L0 即时反馈 | 按钮点击、开关切换 | 0-100ms | easeOut | 按钮按下缩放 0.97 |
| L1 微交互 | 悬停、焦点、状态变化 | 100-300ms | spring | 卡片浮起、输入框聚焦 |
| L2 过渡动画 | 面板展开、视图切换 | 300-500ms | spring | 面板滑入、页面切换 |
| L3 场景转换 | 模态框、全屏切换 | 400-700ms | easeInOut | 对话框出现、主题切换 |
| L4 叙事动画 | 加载、首次渲染 | 500-1500ms | stagger + spring | 时间轴加载、事件交错出现 |

#### 主题切换（View Transitions API）

```tsx
// useThemeStore 中使用 View Transitions API 实现圆形扩散切换
const toggleTheme = async (event: React.MouseEvent) => {
  const x = event.clientX;
  const y = event.clientY;
  const endRadius = Math.hypot(
    Math.max(x, window.innerWidth - x),
    Math.max(y, window.innerHeight - y)
  );

  if (!document.startViewTransition) {
    setTheme(newTheme);
    return;
  }

  const transition = document.startViewTransition(() => {
    setTheme(newTheme);
  });

  await transition.ready;
  document.documentElement.animate(
    { clipPath: [`circle(0px at ${x}px ${y}px)`, `circle(${endRadius}px at ${x}px ${y}px)`] },
    { duration: 500, easing: 'cubic-bezier(0.4, 0, 0.2, 1)', pseudoElement: '::view-transition-new(root)' }
  );
};
```

```tsx
// 成功：底部状态栏闪烁绿光

  animate={saveStatus === 'success' ? { backgroundColor: ['#ffffff', '#2ed573', '#ffffff'] } : {}}
  transition={{ duration: 0.3 }}
>
  {saveStatus === 'success' ? '✓ 已保存' : '保存中...'}
</motion.div>

// 失败：错误图标抖动
<motion.div
  animate={saveStatus === 'error' ? { x: [-4, 4, -4, 4, 0] } : {}}
  transition={{ duration: 0.3 }}
>
  ⚠ 保存失败
</motion.div>
```

#### 拖拽反馈

```tsx
// 使用 Framer Motion 的 drag 实现
<motion.div
  drag
  dragSnapToOrigin
  onDragStart={() => setIsDragging(true)}
  onDragEnd={() => setIsDragging(false)}
  animate={isDragging ? { scale: 1.05, opacity: 0.9 } : { scale: 1, opacity: 1 }}
  whileDrag={{ scale: 1.05, shadow: '0 8px 32px rgba(0,0,0,0.2)' }}
  transition={{ type: 'spring', stiffness: 300, damping: 30 }}
>
  {cardContent}
</motion.div>
```

### 5.3 专注模式动效

重构后用 `useUIStore.focusMode` 替代旧的视觉模式。专注模式（小黑屋）的动效规则：

- 进入专注模式：主画布全屏化，侧栏/工具栏/状态栏淡出（opacity 0, 300ms）
- 退出专注模式：UI 元素淡入恢复
- 专注模式内：保留必要的写作反馈（字数统计、时间显示）
- 使用 Framer Motion 的 layout 动画实现平滑过渡

## 六、主题系统设计

### 6.1 6 主题定义（2026-06-17 重构后）

2026-06-17 重构后扩展为 **6 主题**，每个主题对应不同写作场景与生理需求：

| 主题 ID | 名称 | 模式 | 配色用途 | 适用场景 |
|--------|------|------|---------|---------|
| `paper` | 素纸亮色 | 亮色 | 暖白底（#FAF8F5）+ 墨色文字 + 靛青强调 | 默认主题；日间创作、长时间阅读 |
| `ink` | 墨黑暗色 | 暗色 | 深灰底（#161616）+ 暖白文字 + 亮靛青 | 夜间创作、专注模式 |
| `green` | 护眼绿 | 亮色 | 淡绿调背景，柔和对比 | 长时间写作，缓解视觉疲劳 |
| `parchment` | 羊皮纸 | 亮色 | 米黄复古调，温暖文学感 | 文学/历史/古风题材沉浸创作 |
| `contrast` | 高对比 | 高对比 | 纯黑/纯白 + 强彩色强调 | 无障碍、强光环境、视障辅助 |
| `system` | 跟随系统 | 自适应 | 监听 `prefers-color-scheme` 自动切换 paper/ink | 与系统主题保持一致 |

**设计原则**：
- 6 主题覆盖日间/夜间/护眼/复古/无障碍/系统跟随等场景
- 通过语义色（事件/伏笔/角色）在所有主题间保持一致的视觉语言
- 每套主题独立定义 CSS 变量集（背景层 / 文字层 / 交互层 / 语义色）
- 专注模式（focusMode）作为补充，与主题系统正交

### 6.2 主题切换体验

主题选择器 `src/components/settings/ThemeSelector.tsx`（已嵌入 ShortcutSettings 顶部）：
- 网格预览每个主题（缩略色块 + 名称）
- 选中主题高亮当前激活态
- 点击切换通过 View Transitions API 实现圆形扩散动画（500ms）：从点击位置开始，新主题颜色向外扩展
- `system` 主题会动态监听 `matchMedia('(prefers-color-scheme: dark)')` 的 `change` 事件
- 不支持 View Transitions API 的浏览器降级为即时切换

### 6.3 CSS 变量格式

每个主题在 `src/index.css` 中通过 `[data-theme="<id>"]` 选择器定义独立变量集，使用 RGB 通道格式：

```css
[data-theme="paper"]      { --background: 250 248 245; --foreground: 26 26 26; --primary: 59 91 219; /* ... */ }
[data-theme="ink"]        { --background: 22 22 22;    --foreground: 232 230 225; --primary: 107 138 255; /* ... */ }
[data-theme="green"]      { /* 淡绿调 */ }
[data-theme="parchment"]  { /* 米黄复古 */ }
[data-theme="contrast"]   { /* 高对比 */ }
/* system 通过 JS 动态切换为 paper 或 ink */
```

在 Tailwind 中使用：`bg-background`、`text-foreground`、`bg-primary` 等。

## 七、数据可视化设计

### 7.1 故事弧线图

- **形式**：SVG 曲线图，横轴=时间/章节，纵轴=情感强度
- **交互**：
  - 悬停显示具体情节描述
  - 点击跳转到对应事件
  - 多条弧线叠加对比不同角色/线索
- **动画**：弧线从左到右绘制动画（Framer Motion + stroke-dashoffset），2000ms
- **颜色**：不同角色/线索用不同颜色

### 7.2 角色关系图

重构后由 `RelationshipView` 组件实现（已用新 store + d3）：

- **形式**：力导向图（D3.js force-directed graph）
- **节点**：角色头像 + 名字，大小=出场频率
- **连线**：关系类型用不同颜色和线型
  - 亲情：实线，暖色
  - 友情：实线，蓝色
  - 对立：虚线，红色
  - 暗恋：点线，紫色
- **交互**：
  - 拖拽节点调整布局
  - 悬停节点高亮其所有关联
  - 点击节点展开角色详情
  - 双指缩放
- **动画**：初始布局有力模拟动画，节点弹性就位

### 7.3 情节密度热力图

- **形式**：时间轴上的色块热力图
- **颜色**：冷色（稀疏）→ 暖色（密集）
- **交互**：悬停显示该时段的事件数量和列表
- **动画**：热力图从左到右渐显

### 7.4 伏笔生命周期图

- **形式**：Sankey 图或流程图
- **节点**：伏笔状态（种植→发展→回收/废弃）
- **连线**：伏笔流转路径，宽度=伏笔数量
- **交互**：悬停显示具体伏笔列表
- **动画**：流动粒子沿连线方向移动

## 十、技术实现建议

### 10.1 动画技术选型（重构后）

| 场景 | 技术 | 理由 |
|------|------|------|
| 声明式组件动画 | **Framer Motion 11**（已集成） | React 原生、声明式、弹簧物理、AnimatePresence |
| 主题切换过渡 | View Transitions API | 浏览器原生、圆形扩散效果 |
| 滚动驱动动画 | Framer Motion useScroll + useTransform | 时间轴缩放时的视差效果 |
| 力导向图 | D3.js force simulation | 角色关系图标准方案 |
| SVG 路径动画 | Framer Motion + stroke-dashoffset | 故事弧线绘制 |
| 手势/拖拽 | Framer Motion drag | 事件拖拽排序 |

> ⚠️ 重构后唯一动画引擎为 Framer Motion 11。旧组件中的 anime.js v4 引用待迁移完成后移除。

**Framer Motion 基本用法**：
```tsx
import { motion, AnimatePresence } from 'framer-motion';

// 基础动画
<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  exit={{ opacity: 0, y: -20 }}
  transition={{ duration: 0.3, ease: 'easeOut' }}
>
  内容
</motion.div>

// 弹簧动画
<motion.div
  animate={{ scale: 1 }}
  transition={{ type: 'spring', stiffness: 80, damping: 10 }}
>
  内容
</motion.div>

// 列表交错
const containerVariants = {
  hidden: { opacity: 0 },
  visible: { opacity: 1, transition: { staggerChildren: 0.05 } }
};
const itemVariants = {
  hidden: { opacity: 0, y: 20 },
  visible: { opacity: 1, y: 0 }
};
```

## 十一、功能联动（跨视图协同）

2026-06-17 重构后实现 **5 个跨视图联动点**，让不同视图/面板形成有机整体（对齐通用协同写作体验）：

| # | 触发动作 | 目标响应 | 实现机制 |
|---|---------|---------|---------|

## 十二、细纲演进图（对齐"洛笙创作"）

### 12.1 背景与场景
小说创作过程中，细纲会经历多次大幅修改（拆/合/换序/重写）。创作者需要：
- 回看每一版细纲快照，对比哪些事件被增删/调整
- 在新方向走偏时**回滚**到某个历史版本
- 长期演进中保留思考脉络

### 12.2 数据模型
- 新增表 `outline_versions(id / workspaceId / content / description / createdAt)`
- `content` 为 JSON 快照（包含轨道分组的事件列表）
- 防抖：5 分钟最多自动保存一次（避免频繁 IO）

### 12.3 交互设计
- **入口**：OutlineView 顶部按钮"演进历史"
- **展示形式**：TDesign Drawer 从右侧弹出，时间线列表（最新版本在上）
- **每条版本卡片**：时间戳 / 描述（用户可在创建时自定义）/ 事件总数 / 操作按钮
- **Diff 对比**：左右双栏布局，按轨道分组列出事件标题，颜色标记新增 / 删除 / 修改
- **回滚**：点击"恢复此版本" → 二次确认 → 通过 `useUpdateEvent` 批量更新
- **删除**：版本可单独删除（不影响其他版本）

### 12.4 API
- `GET    /api/workspaces/:wsId/outline-versions` 列表
- `GET    /api/workspaces/:wsId/outline-versions/:id` 详情
- `POST   /api/workspaces/:wsId/outline-versions` 创建（防抖 5min）
- `POST   /api/workspaces/:wsId/outline-versions/:id/restore` 回滚
- `DELETE /api/workspaces/:wsId/outline-versions/:id` 删除

## 十三、伏笔呼应链条（对齐"洛笙创作"）

### 13.1 背景与场景
"伏笔"在小说中通常是**多对多**的呼应网络（一个伏笔可能由多个前置事件铺垫，又同时影响多个后续回收点）。传统的列表/线性视图无法直观呈现这种关系。

### 13.2 数据模型
- `foreshadowings` 表新增字段 `relatedForeshadowingIds`（JSON 字符串数组，存储关联伏笔 ID）
- 后端路由 `validation.ts` 增加对应 schema
- 通过自关联实现多对多

### 13.3 视图切换
ForeshadowingPanel 顶部提供 **两种视图模式**：
1. **列表视图**：原有的卡片式列表（标题 / 状态 / 描述 / 关联事件）
2. **呼应链条图谱**：D3 force-directed graph

### 13.4 图谱设计（ForeshadowingGraph）
- **节点配色按状态**：
  - `planted`（已埋）→ 黄色
  - `developed`（发展中）→ 蓝色
  - `resolved`（已回收）→ 绿色
  - `abandoned`（已废弃）→ 灰色
- **节点 size 按 degree 动态**：连接越多越大（16-36px）
- **连线**：带箭头 marker，方向为「前置 → 回收」
- **交互**：
  - 单击节点 → 高亮自身与所有邻居（其他节点 `opacity: 0.3`）
  - 双击节点 → 跳转到伏笔编辑表单
  - 拖拽节点 → 调整布局（D3 力模拟）

### 13.5 编辑表单
EventEditorDialog 中的伏笔编辑表单新增"关联伏笔"字段：
- TDesign `Select` 组件（`multiple` + `filterable`）
- 数据源为当前工作区的全部伏笔
- 选中后双向更新（保存时同步对方的 `relatedForeshadowingIds`）
