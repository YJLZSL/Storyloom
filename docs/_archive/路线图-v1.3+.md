# Storyloom v1.3.0+ 后续方向规划

> 创建日期：2026-06-21
> 当前版本：v1.3.0
> 状态：草案（待讨论确认）

## 一、项目现状评估

### 已完成的核心能力
- 时间轴视图（含 7 种视图模式：时间轴/大纲/叙事/甘特/树状/统计/关系图）
- 角色系统、伏笔系统、世界观设定、关联线
- 书签系统、地图系统（v1.2.0 新增）
- AI 助手（7 个提供商接入）
- 8 套主题（洛圣/子夜/森林/水墨/高对比/桜/深海/极光）
- 专注模式、Zen 模式、番茄钟
- 命令面板、快捷键系统
- 自动更新、数据库备份

### 已知技术债
- `OutlineView.tsx`（948 行）待拆分为子组件
- `EntityPanel<T>` 泛型组件待抽取（重复代码）
- 测试覆盖率极低（仅 12 个测试文件，193 个用例但覆盖不全）
- 死代码：`CalendarConfigDialog.tsx`（252 行）未被任何模块 import
- `ui/` 下 8 个 shadcn 组件未使用（checkbox/command/popover/select/sheet/slider/switch/tabs）
- 仓库冗余约 2.7GB（旧版本构建产物）

## 二、v1.3.0 — AI 深度集成 + 数据持久化（建议优先）

### 目标
让 AI 助手从"通用对话"升级为"了解你的故事的创作伙伴"。

### 功能点

**2.1 AI 对话历史持久化**
- 后端实现 `ai_conversations` 和 `ai_cache` 表的完整 CRUD 路由
- 前端 `AIPanel.tsx` 对接持久化 API（替换当前内存状态）
- 对话列表侧边栏：历史对话浏览、搜索、删除
- 对话上下文自动注入工作区数据（角色/事件/伏笔摘要）

**2.2 AI 工作区上下文读取**
- 前端调用已就绪的 `/api/ai/workspace-context` 端点
- 自动提取当前工作区的故事摘要作为 AI system prompt
- 支持手动编辑上下文注入范围（选择哪些角色/事件参与对话）

**2.3 AI 辅助创作增强**
- 基于时间轴上下文的事件描述续写/扩展
- 角色对话生成（根据角色特征和关系）
- 伏笔回收建议（检测未回收的伏笔并建议回收方式）
- 故事一致性检查（检测时间矛盾、角色行为不一致等）

### 预估工作量
- 后端 CRUD：约 1-2 天
- 前端对接 + UI：约 2-3 天
- AI 增强功能：约 3-5 天

## 三、v1.4.0 — 视觉小说引擎对接 + 导出

### 目标
打通从"故事设计"到"可运行的视觉小说"的最后一公里。

### 功能点

**3.1 WebGAL 脚本导出**
- 将时间轴事件 + 角色对话导出为 WebGAL 兼容脚本
- 支持场景切换、角色立绘指令、BGM 指令映射
- 导出前预览生成的脚本

**3.2 ZIP 完整导出/导入**
- 整个工作区打包为 ZIP（含事件/角色/伏笔/书签/地图/资产文件）
- 支持跨设备迁移和备份
- 导入时自动处理 ID 冲突

**3.3 JSON 结构化导出**
- 导出为标准化 JSON 格式（可被其他工具消费）
- 支持选择性导出（仅事件、仅角色、仅关系图等）

### 预估工作量
- WebGAL 导出：约 3-5 天
- ZIP 导出/导入：约 2-3 天
- JSON 导出：约 1 天

## 四、v1.0.0+ — 视觉体验升级（路线图，部分已合并至正式版）

### 目标
回应用户"美术没什么改变"的反馈，带来更显著的视觉提升。

### 改进方向

**4.1 主题系统增强**
- 主题预览：设置面板中实时预览主题效果（色板 + 组件样例）
- 自定义主题：用户可微调主题色/字体/纹理并保存为自定义预设
- 季节性主题自动推荐（可选）

**4.2 微交互升级**
- 时间轴拖拽动画优化（弹性回弹 + ghost 预览）
- 事件卡片 hover 信息浮层（显示关联角色/伏笔摘要）
- 面板切换过渡动画（滑入/淡入编排）
- 操作成功/失败的微动画反馈

**4.3 响应式优化**
- 小屏幕 StatusBar 布局修复
- 面板宽度拖拽调整（而非固定宽度）
- 时间轴缩放级别记忆（每个工作区独立保存）

### 预估工作量
- 主题增强：约 2-3 天
- 微交互升级：约 3-5 天
- 响应式优化：约 2 天

## 五、v1.6.0+ — 远期方向（按需选择）

以下为远期可选方向，根据用户反馈和优先级灵活调整：

**5.1 全文搜索与替换**
- 后端 `/api/workspaces/:workspaceId/search` 已定义 schema（validation.ts 中 searchQuery/replaceBody）
- 需实现后端路由 + 前端搜索面板 UI
- 支持范围过滤（仅事件/仅角色/仅伏笔）

**5.2 版本历史（时光机）**
- `revisions` 表已创建，后端路由已实现
- 前端需实现版本历史面板 + diff 视图 + 回滚操作

**5.3 多语言支持扩展**
- 当前仅 zh-CN / en-US
- 可扩展日语（视觉小说核心用户群）

**5.4 插件系统**
- 允许用户安装第三方扩展（主题包/导出格式/AI 模型）
- 需要设计插件 API + 安全沙箱

## 六、技术债清理计划

建议在每个版本中穿插清理以下技术债：

| 优先级 | 项目 | 说明 |
|--------|------|------|
| 高 | 拆分 OutlineView | 948 行单文件，难以维护 |
| 高 | 增加测试覆盖 | 当前仅 193 用例，核心流程缺少 E2E |
| 中 | 抽取 EntityPanel<T> | 减少角色/伏笔/世界观面板的重复代码 |
| 中 | 删除死代码 | CalendarConfigDialog + 8 个未用 shadcn 组件 |
| 低 | 清理仓库冗余 | 旧版本构建产物约 2.7GB |
| 低 | P2 自动更新引导 | 旧版本用户手动下载提示 |
