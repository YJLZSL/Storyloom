## 2. 剩余后端开发任务

### 2.1 WebGAL 导出器（v1.3 - P1 - 大）

依赖 §2.1–§2.5（全部已完成）。

**位置**：新建 [`server/services/exporters/webgal.ts`](../server/services/exporters/webgal.ts)。

**输入**：完整工作区 IR（scenes + beats + choices + flags + assets 的内存表示）。

**输出**：一个完整可被 WebGAL 编辑器打开的项目目录（zip）：
```
<exported>/
├── game/
│   ├── scene/
│   │   └── start.txt          ← 入口；按 scenes.sceneOrder 串联
│   │   └── <sceneName>.txt    ← 一个 scene 一个文件
│   ├── figure/<filename>.png  ← portrait_asset 拷过来
│   ├── background/<filename>  ← scene background
│   └── bgm/<filename>         ← scene bgm
└── README-storyloom.txt
```

**Beat → WebGAL 语句翻译**：
| Storyloom Beat | WebGAL 行 |
|---|---|
| line（角色 X 说 "..."）| `X:...; ` |
| line（旁白）| `:...;` |
| changeBg（来自 scene 切换）| `changeBg:bg.png -next;` |
| changePortrait | `changeFigure:fig.png -left -next;` |
| choice | `choose:选项A:next1.txt|选项B:next2.txt;` |
| sfx / anim | `playEffect:xxx.ogg;` / `setAnimation:bounce -target=fig-left;` |

**端点**：`POST /api/workspaces/:id/export/webgal` 直接返回 zip 流。

**验收**：把导出的项目目录解压，丢进 WebGAL 编辑器能正常加载，主线可一路跑通。

### 2.2 全文搜索 + 替换（v1.4 - P0 - 中）

依赖 §2.1–§2.7（全部已完成）。

- `GET /api/workspaces/:id/search?q=&scope=event,beat,outline,character,worldsetting,foreshadowing,scene` 返回每个 hit 的 `{entityType, entityId, snippet, matchCount}`
- `POST /api/workspaces/:id/replace`（body: `{q, replacement, scope, dryRun}`）事务化批量替换；dryRun 先返回 will-change 列表

实现从 LIKE %q% 起步，未来再换 SQLite FTS5。

### 2.3 内置试玩预览后端（v1.4 - P1 - 中）

依赖 §2.1–§2.7（全部已完成）。

`GET /api/workspaces/:id/preview/script` 返回扁平化 IR JSON：
```json
{
  "version": "1.0",

    {"id": "...", "name": "...", "background": {...}, "bgm": "...", "beats": [...]}

  "flags": [...],
  "entrySceneId": "..."
}
```

前端 pixi 渲染器直接吃这份 JSON 就能播。

## 3. 剩余后端 Bug 修复

### P1 — 健壮性

1. **6 处子资源 POST 缺 workspace 存在性校验**（§3.P1-6）
   - 涉及：events / tracks / characters / world-settings / connections 的 POST（scenes/flags/maps/assets 的新路由已在 v1.2 中加入校验）
   - 修复：抽 `validateWorkspace(workspaceId)` 公共函数到 `server/lib/validation.ts`，每个子资源 POST 入口先调用

2. **events 搜索仅匹配 title**（§3.P1-7）
   - 文件：`server/routes/events.ts`
   - 修复：扩展为 `(title LIKE ? OR summary LIKE ? OR location LIKE ?)`，用 `escapeLike` 转义

3. **events 分页未 clamp**（§3.P1-8）
   - 修复：`limit = Math.min(Math.max(limit ?? 50, 1), 200)`、`offset = Math.max(offset ?? 0, 0)`

4. **server/index.ts 无 graceful shutdown / 动态 import 健康检查 / isDirectRun 脆弱**（§3.P1-9）
   - 修复：① 监听 SIGTERM/SIGINT，调用 `app.close()` + `closeDb()` 后再 `process.exit(0)`；② `/health` 端点改为同步用已注入的 `app.db`；③ `isDirectRun` 用 `import.meta.url === pathToFileURL(process.argv[1]).href` 替换字符串比对

### P2 — 一致性

5. **db/index.ts closeDb 无 try/catch；DATA_DIR mkdirSync 无异常处理**（§3.P2-12）
   - 修复：包 try/catch + log

6. **events 批量操作字段缺失**（§3.P0-3）
   - 文件：`server/routes/events.ts` 的 `POST /:workspaceId/events/batch`
   - 现象：批量 update 漏掉 trackId / startTime / endTime / narrativeOrder / characterIds / worldSettingIds；批量 create 漏 narrativeOrder
   - 修复：批量 schema 与单条 update / create schema 字段对齐

## 8. 已知陷阱

1. **Tailwind 是 v4 不是 v3** — 用 `@theme` CSS-first 而非 `tailwind.config.js`
2. **shadcn/ui 未集成** — 组件在 `src/components/ui/` 但未经 `npx shadcn add` 正式安装
3. **anime.js v4 API 与 v3 不同** — 已迁移到 Framer Motion 11
4. **全 ESM 模块** — 不能用 `require()`，import 必须带 `.js` 后缀
5. **archiver v8 是 ESM** — 用 `import { ZipArchive } from 'archiver'` 而非 `archiver('zip')`
6. **drizzle-kit 没有 db:generate 脚本** — 用 `npx drizzle-kit generate`
7. **后端 route 参数**：foreshadowings 用 `foresId`，connections 用 `connId`
8. **不要改 appId / GitHub 仓库 URL** — v2.0 才动
9. **任何 SQL 都走 Drizzle ORM** — 不要 raw SQL 绕过
