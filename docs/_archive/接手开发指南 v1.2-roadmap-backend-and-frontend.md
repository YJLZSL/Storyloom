## 2. 后端路线图开发任务清单

> **总原则**：每个任务 = 一组路由文件 + 一张/几张表 + 一组 Schema + 一段 curl 烟测。完成后单独 commit，`npm run typecheck` 与 `npm run build:server` 都要通过。任务 2.1（数据模型）是其它一切的前置。

### 2.1 视觉小说数据模型扩展（v1.2 - P0 - 大）

**背景**：路线图 §1.2.1 已规定 Scene/Beat/Choice/Flag 四对象 + 资产关联表。

**数据模型变更**：在 [`server/db/schema.ts`](../server/db/schema.ts) 末尾新增 8 张表。所有 id 用 `text('id').primaryKey()` + `nanoid()`；外键全部 `onDelete: 'cascade'`；时间戳用 `integer('created_at', { mode: 'timestamp' }).notNull().$defaultFn(() => new Date())`。

| 表 | 关键字段 |
|---|---|
| `scenes` | id / workspaceId(FK) / name / backgroundAssetId(FK assets 可空) / bgm(text) / sceneOrder(int) / mapId(FK maps 可空) / settingsJson |
| `beats` | id / sceneId(FK) / kind(`line`/`choice`/`jump`/`sfx`/`anim`) / characterId(FK characters 可空) / portraitAssetId(FK assets 可空) / text / metadata(json) / beatOrder(int) |
| `choices` | id / beatId(FK) / label / nextSceneId(FK scenes 可空) / condition(text 描述 Flag 表达式) / choiceOrder(int) |
| `flags` | id / workspaceId(FK) / name(unique per workspace) / defaultValue(json) / description |
| `assets` | id / workspaceId(FK) / kind(`avatar`/`portrait`/`scene`/`map`/`bgm`/`sfx`) / fileName / mimeType / fileSize / sha256(unique) / width / height / metadata(json) |
| `character_assets` | characterId(FK) / assetId(FK) / role(`avatar`/`portrait_default`/`portrait_smile`...) / displayOrder ; PK = (characterId, assetId, role) |
| `event_assets` | eventId(FK) / assetId(FK) / role(`scene_thumb`/`reference`) ; PK = (eventId, assetId, role) |
| `scene_assets` | sceneId(FK) / assetId(FK) / role(`background`/`overlay`/`bgm`) ; PK = (sceneId, assetId, role) |

**Schema 与类型**：
- 在 [`shared/types.ts`](../shared/types.ts) 新增 `Scene` / `Beat` / `Choice` / `Flag` / `Asset` 接口，与 schema 字段一一对应
- 在 [`server/lib/validation.ts`](../server/lib/validation.ts) 新增对应 `sceneIdParam` / `beatIdParam` / `choiceIdParam` / `flagIdParam` / `assetIdParam` 与 `createSceneBody` / `updateSceneBody` 等 Fastify JSON Schema

**迁移**：跑 `npm run db:generate` 让 drizzle-kit 产 `drizzle/0xxx_add_visual_novel_tables.sql`；启动时 `migrate.ts` 自动应用；要写一个**前向兼容**的 noop 迁移检查（确保 v1.1.x 客户端打开 v1.2 数据库不会崩，反之同样）。

**验收**：`npm run typecheck` 通过；本地 `data/dev.db` 能看到 8 张新表；`npm run db:studio` 能打开。

### 2.3 图片资产存储（v1.2 - P0 - 大）

依赖 §2.1。

**物理文件**：放到 `app.getPath('userData') + '/assets/' + workspaceId + '/' + sha256.slice(0,2) + '/' + sha256` —— SHA-256 内容寻址，自动去重。

**上传**：`POST /api/workspaces/:workspaceId/assets` 走 `@fastify/multipart`（**新依赖**：`npm i @fastify/multipart`），单文件 ≤ 50 MB；后端计算 SHA-256；如果 `sha256` 已存在则不写盘只复用；返回 `{id, sha256, fileName, kind, width, height, fileSize}`。

**读取**：
- `GET /api/assets/:id/file` — 直接 stream 物理文件（带 ETag = sha256）
- `GET /api/assets/:id/thumbnail?w=120&h=120` — 用 sharp（**新依赖**：`npm i sharp`）生成缩略图，缓存到 `userData/assets/.thumbs/<sha256>_<w>x<h>.webp`

**关联**：`POST /api/characters/:id/assets`（绑头像/立绘）、`POST /api/scenes/:id/assets`（绑背景/BGM）、`POST /api/events/:id/assets`（绑场景缩略图）。

**删除**：删 asset 行 + 物理文件（仅当无其它行引用同 sha256 时；多对多由 character_assets / event_assets / scene_assets 三张表跟踪）。

**验收**：能上传一张 PNG → `assets/` 目录里能看到文件 → `GET /api/assets/:id/file` 浏览器可见图片 → `GET .../thumbnail?w=120` 返回 webp。

[`server/routes/maps.ts`](../server/routes/maps.ts) 全 CRUD。

**新依赖**：`npm i archiver`。

扩展 [`server/routes/workspaces.ts`](../server/routes/workspaces.ts) 的 `GET /:workspaceId/export`（已有 JSON 出口，不要破坏旧契约；新增 `?format=zip`）：

- zip 内容：`workspace.json` / `events.json` / `tracks.json` / `characters.json` / `world-settings.json` / `foreshadowings.json` / `connections.json` / `outline-versions.json`（**别再漏，参见 §3 第 5 条**）/ `scenes.json` / `beats.json` / `choices.json` / `flags.json` / `maps.json` / `assets/<sha256>` 物理文件 / `README.md`（schema 描述 + Storyloom 版本）

### 2.6 WebGAL 导出器（v1.3 - P1 - 大）

依赖 §2.1–§2.5。

**位置**：新建 [`server/services/exporters/webgal.ts`](../server/services/exporters/webgal.ts)。

**输入**：完整工作区 IR（scenes + beats + choices + flags + assets 的内存表示）。

**输出**：一个完整可被 [WebGAL 编辑器](https://github.com/MakinoharaShoko/WebGAL) 打开的项目目录（zip）：
```
<exported>/
├── game/
│   ├── scene/
│   │   └── start.txt          ← 入口；按 scenes.sceneOrder 串联
│   │   └── <sceneName>.txt    ← 一个 scene 一个文件
│   ├── figure/<filename>.png  ← portrait_asset 拷过来
│   ├── background/<filename>  ← scene background
│   └── bgm/<filename>         ← scene bgm
└── README-storyloom.txt
```

**Beat → WebGAL 语句翻译**：
| Storyloom Beat | WebGAL 行 |
|---|---|
| line（角色 X 说 "..."）| `X:...; ` |
| line（旁白）| `:...;` |
| changeBg（来自 scene 切换）| `changeBg:bg.png -next;` |
| changePortrait | `changeFigure:fig.png -left -next;` |
| choice | `choose:选项A:next1.txt|选项B:next2.txt;` |
| sfx / anim | `playEffect:xxx.ogg;` / `setAnimation:bounce -target=fig-left;` |

**端点**：`POST /api/workspaces/:id/export/webgal` 直接返回 zip 流。

**验收**：把导出的项目目录解压，丢进 WebGAL 编辑器（或 webview 试玩）能正常加载，主线可一路跑通。

## 3. 后端打磨与排错任务（在 §2 全部完成后做）

> 上一轮我已系统过 read 所有后端文件，识别出以下 P0/P1/P2 bug。请逐个修复，每条单独 commit，commit 信息含 issue 描述与回归用例。

### P0 — 完全破坏性

1. **PATCH/DELETE 参数名不匹配**
   - 文件：[`server/lib/validation.ts`](../server/lib/validation.ts)
   - 现象：`foreshadowingIdParam` 的 JSON Schema 声明 `properties: { foreshadowingId }, required: ['foreshadowingId']`，但实际路由用 `:foresId`；`connectionIdParam` 同理用 `:connId`。Ajv 校验直接 404 / 422。
   - 修复：把 schema 里的 `foreshadowingId` → `foresId`，`connectionId` → `connId`（或反之改路由）。**二选一保持一致即可**。
   - 回归：`curl -X DELETE http://localhost:3001/api/workspaces/<wsId>/foreshadowings/<foresId>` 返回 200。

2. **workspaces PATCH 漏 settingsJson 字段**
   - 文件：[`server/routes/workspaces.ts`](../server/routes/workspaces.ts)
   - 现象：前端能 PATCH name / description / coverColor，唯独 settingsJson（包括用户偏好、视图状态）不会写入。
   - 修复：把 settingsJson 加进 `updateWorkspaceBody` schema 与 update set 子句。

3. **events 批量操作字段缺失**
   - 文件：[`server/routes/events.ts`](../server/routes/events.ts) 的 `POST /:workspaceId/events/batch`
   - 现象：批量 update 漏掉 trackId / startTime / endTime / narrativeOrder / characterIds / worldSettingIds；批量 create 漏 narrativeOrder。
   - 修复：批量 schema 与单条 update / create schema 字段对齐。
   - 回归：批量 update 一次性挪 5 个事件到新轨道 + 改时间，全部生效。

4. **foreshadowings POST 没绑 schema validation**
   - 文件：[`server/routes/foreshadowings.ts`](../server/routes/foreshadowings.ts)
   - 现象：POST 直接 `request.body as any`，不做校验，前端传脏数据会写库。
   - 修复：改用 `app.withTypeProvider<TypeBoxTypeProvider>().post('/', { schema: { body: createForeshadowingBody } }, ...)`。

5. **outlineVersions 在 export / import / auto-save 恢复中缺失 → 数据丢失**
   - 文件：[`server/routes/workspaces.ts`](../server/routes/workspaces.ts)（export/import 端点）+ [`server/services/auto-save.ts`](../server/services/auto-save.ts)（recoverFromAutoSave）+ [`shared/types.ts`](../shared/types.ts) 的 `ExportData`
   - 现象：用户用过大纲版本管理后，导出再导入工作区，所有 outline-versions 历史记录丢失；自动备份恢复同样丢。
   - 修复：① ExportData 加 `outlineVersions: OutlineVersion[]` 字段；② export 端点查 + 写；③ import 端点读 + 插；④ recoverFromAutoSave 一并还原 outlineVersions；⑤ auto-save 快照 schema 同步更新；⑥ 同时更新 [`server/services/auto-save.ts::recoverFromAutoSave`](../server/services/auto-save.ts) 中**还要把 workspaces.updatedAt 刷新**。

### P1 — 健壮性

6. **6 处子资源 POST 缺 workspace 存在性校验**
   - 涉及：events / tracks / characters / world-settings / foreshadowings / connections 的 POST
   - 现象：传一个不存在的 workspaceId 时不报 404 而是直接在 events 表写出孤儿行（FK 没启 PRAGMA 时）
   - 修复：抽 `validateWorkspace(workspaceId)` 公共函数到 [`server/lib/validation.ts`](../server/lib/validation.ts)，每个子资源 POST 入口先调用。

7. **events 搜索仅匹配 title**
   - 文件：[`server/routes/events.ts`](../server/routes/events.ts)
   - 修复：扩展为 `(title LIKE ? OR summary LIKE ? OR location LIKE ?)`，用 `escapeLike` 转义。

8. **events 分页未 clamp**
   - 现象：limit=10000 直接打数据库；offset 为负不报错。
   - 修复：`limit = Math.min(Math.max(limit ?? 50, 1), 200)`、`offset = Math.max(offset ?? 0, 0)`。

9. **server/index.ts 无 graceful shutdown / 动态 import 健康检查 / isDirectRun 脆弱**
   - 文件：[`server/index.ts`](../server/index.ts)
   - 修复：① 监听 SIGTERM/SIGINT，调用 `app.close()` + `closeDb()` 后再 `process.exit(0)`；② `/health` 端点改为同步用已注入的 `app.db`，不要再 `await import('./db/index.js')`；③ `isDirectRun` 用 `import.meta.url === pathToFileURL(process.argv[1]).href` 替换字符串比对。

10. **error-handler.ts JSON 解析错误返回 500**
    - 文件：[`server/plugins/error-handler.ts`](../server/plugins/error-handler.ts)
    - 修复：识别 `error.code === 'FST_ERR_CTP_INVALID_JSON_BODY'` 或 `error.statusCode === 400` 时，返回 400 + `{error: 'invalid_json', message}`。

### P2 — 一致性

11. **备份文件名前缀仍是 `timeline-creator-`**
    - 文件：[`server/db/migrate.ts`](../server/db/migrate.ts) + [`server/services/auto-save.ts`](../server/services/auto-save.ts)

## 4. 已知前端缺陷与重设大方向

> 这是给视觉 AI（或后端 AI 完成 §2/§3 后自己接）用的"问题清单 + 修复方向"，**不写到具体代码 diff**。

### F-1 缺少「设置」入口

**现状**：[`src/components/layout/TopToolbar.tsx`](../src/components/layout/TopToolbar.tsx) 只有工作区下拉、缩放、新建事件、保存、命令面板、语言、主题；用户找不到偏好、教程、关于、API Key 配置、备份位置、数据迁移按钮。

**大方向**：
- 在 TopToolbar 右侧（语言/主题之间或之前）加 `Settings`（齿轮图标，lucide-react `Settings`）按钮
- 点击打开 `<SettingsDialog>`（沿用 shadcn Dialog 大尺寸 max-w-3xl 或独立 SettingsView）
- 5 个 Tab：**偏好** / **主题** / **快捷键**（已有 ShortcutSettings 可复用）/ **教程**（详见 F-5）/ **关于与数据**（版本号、userData 路径、自动备份位置、清空缓存、导出诊断日志）
- 设置项持久化到 `useUIStore` 的 settingsJson 字段或独立 `useSettingsStore`

### F-2 命令面板（Ctrl+K）打开后透明 + 黑屏

**现状**：[`src/components/ui/dialog.tsx`](../src/components/ui/dialog.tsx) 的 `DialogOverlay` 是 `fixed inset-0 z-50 bg-black/80`；[`src/components/ui/command.tsx`](../src/components/ui/command.tsx) 的 `CommandDialog` 把 cmdk Command 套在 `DialogContent` 内（`bg-background p-0`），Command 自身又是 `bg-popover`。在某些主题下 popover 与 background 同色或者 popover 失效就会看到"框透明 + 后面黑屏"。同时 cmdk 要求 `<DialogTitle>` 否则 Radix a11y 会警告。

**修复方向**：
1. 检查所有 6 主题（[`src/index.css`](../src/index.css)）的 `--popover` 与 `--popover-foreground` 是否每个主题都明确赋值（**非透明、非 var 引用错误**）
2. 给 `CommandDialog` 显式样式：`<DialogContent className="bg-popover text-popover-foreground border shadow-2xl backdrop-blur p-0 top-[20%] translate-y-0 max-w-2xl">`（提到屏幕上方 20% 处，最大宽度收紧）
3. 加 `<DialogTitle className="sr-only">命令面板</DialogTitle>` 解决 a11y 警告
4. 把 `bg-popover` 强制为不透明（去 `/N` 透明数）
5. 6 主题分别截图入 `docs/screenshots/v1.2-after/command-palette-<theme>.png`

### F-3 缩放（Zoom）粗糙

**现状**：[`TopToolbar.tsx`](../src/components/layout/TopToolbar.tsx) 仅 `zoomIn` / `zoomOut` 离散按钮；[`useTimelineStore`](../src/stores/useTimelineStore.ts) zoom 是分档；无 Ctrl+滚轮、无 Ctrl+0 重置；卡片字号、轨道高度、时间轴标尺密度在极端 zoom 下不一致；OutlineView 不响应 zoom。

**修复方向**：
1. zoom 改成连续值 `[0.5, 3.0]`，setZoom(v) 自动 clamp
2. 在 TimelineCanvas / OutlineView 监听 Ctrl+滚轮（`e.ctrlKey && e.preventDefault()` 改 zoom 而非滚动）
3. 快捷键：`Ctrl+=` / `Ctrl+-` / `Ctrl+0`（重置到 1.0）
4. 时间轴轨道高度、事件卡片字号、ruler 刻度密度全部用 `var(--zoom)` × 基准值（CSS 自定义属性 + tailwind arbitrary）
5. OutlineView 字号档独立（A- / A+ 按钮在 ContextPanel 顶部）

### F-4 视图与右栏面板协同差

**现状**：[`CommandPalette.tsx`](../src/components/command-palette/CommandPalette.tsx) 行为不统一：搜伏笔/世界观仅 setActivePanel，搜事件切 timeline + scrollToEvent，搜角色仅 setActivePanel('characters') 不切到关系图；切视图后选中的事件不一定在新视图里被高亮 / 滚到。

**修复方向**：
1. 抽离 `useSelectionStore`（事件 / 角色 / 伏笔 / 世界观 / Scene / Choice）
2. 每个视图（Timeline / Outline / Narrative / Gantt / Tree / Stats / Relationship）订阅 selection，在切视图时自动 scroll/focus
3. 命令面板 onSelect 行为统一为 `selectXxx(id) → revealInBestView()`，其中 `revealInBestView` 根据 entityType 选最合适的视图（事件→timeline，角色→relationship 或 characters 面板，伏笔→ForeshadowingBoard）
4. 写一段简短的 selection 协同 spec 到 `.trae/specs/selection-context-coherence/`（视觉 AI 自己起 spec）

### F-5 缺少 in-app 教程

**现状**：所有文档在 `docs/` 仓库内，对终端用户不可见；新用户打开应用不知道时间轴怎么用、AI 面板怎么连 API、命令面板怎么开。

**修复方向**：
1. Settings 的「教程」Tab：左侧目录 + 右侧 react-markdown 渲染（已集成）
2. 教程文件位置：打包到 `public/tutorials/<key>.md`（前端读 `import.meta.glob`），不放后端
3. 11 篇覆盖：`getting-started` / `timeline-view` / `outline-view` / `tree-view` / `relationship-graph` / `script-editor`（v1.2 新视图）/ `branch-map`（v1.2 新视图）/ `ai-panel` / `command-palette` / `themes-and-focus` / `auto-backup-and-export-webgal`
4. 每篇结构：是什么 / 为什么需要 / 怎么用（步骤 + GIF/截图占位）/ 快捷键速查 / 常见问题
5. 教程 markdown 可包含 `![](./images/xxx.png)` 引用 `public/tutorials/images/`

> **本轮不写实际教程文本**——只搭骨架与第一篇 `getting-started.md`；其余每篇随对应功能完成时再写。

## 7. 不做的事（边界）

继承路线图 §2 全部 ❌ 项：
- 不做 macOS / Linux / Web 在线版 / 移动端
- 不做云同步 / 多端协作
- 不做插件市场 / 模板市场 / AI 模型市场
- 不做 AI 生成封面 / 立绘 / 场景图
- **不自研视觉小说引擎**（WebGAL / Ren'Py 已经做得很好，做"上游"不做"下游"）
- 不自研编程语言式剧本 DSL（用表格式即可）
- 不做组织架构图 / 词条字典 / 随机名字生成器
- 不追 Vitest 覆盖率硬指标 / Actions 自动 release

本轮新增边界：
- **不要回退到"小说作家工作台"定位** —— Storyloom 已锁定为「视觉小说编剧上游工作台」；新功能必须问"这件事是否服务于 VN 工作流"
- **不要改 appId / GitHub 仓库 URL**（v2.0 才动；当前会破坏 v1.1.0 客户端的自动更新链）
- **不要写代码 + 路线图同时开搞** —— 先把数据模型 §2.1 写完，所有其它任务都依赖它
