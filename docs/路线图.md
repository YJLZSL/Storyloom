## 一、当前状态（v1.0.0 统一稳定版）

v1.0.0 是项目经历多轮快速迭代后的正式稳定版本，包含此前 v1.1.x 至 v1.5.0 的全部功能。

### 已交付的核心能力

**创作工具**：时间轴视图（7 种模式：时间轴/大纲/叙事/甘特/树状/统计/关系图）、角色系统、伏笔系统、世界观设定、关联线、书签系统、地图系统。

**写作与编辑**：Zen Mode 专注写作、三栏写作布局（章节目录 + 编辑器 + 数据面板）、每日目标字数、番茄钟。

**资料库**：笔记管理、文件夹树、标签系统、快速记录。

**AI 助手**：7 个 AI 提供商接入、对话历史持久化（SQLite）、工作区上下文自动注入、场景化功能卡片。

**视觉体验**：三层渲染模型（交互层/内容层/渲染层）、8 套完整主题、PixiJS 粒子背景、GSAP 动画引擎、LoomSplash 启动动画、页面过渡。

**基础设施**：Tauri 2.x 桌面壳、自动更新（minisign 签名）、NSIS 定制安装程序、命令面板、快捷键系统、中英双语国际化。

### 已知技术债

| 项目 | 严重度 | 说明 |
|------|--------|------|
| OutlineView.tsx 948 行 | 高 | 单文件过大，待拆分为子组件 |
| 测试覆盖不全 | 高 | 193 个用例但缺少 E2E 和集成测试 |
| VN/剧本编辑器前端缺失 | 高 | 后端 API 已就绪，前端 14 hooks + 7 组件未实现 |
| EntityPanel\<T\> 泛型重复 | 中 | 角色/伏笔/世界观面板有大量重复代码 |
| index.css 2921 行 | 中 | 样式文件过大，待按功能模块拆分 |
| 签名密钥冗余 | 低 | 5 套密钥文件，需确认有效密钥并清理 |

### 目标
修复核心 Bug、补全 VN 编辑器前端、提升代码质量。

**2.3 修复图标显示**
- 确认 `icon.ico` 正确嵌入 Windows 二进制资源

### 技术债清理

**2.5 拆分 OutlineView**
- 将 948 行文件拆分为 3-4 个子组件 + 1 个 hooks 文件

**2.6 测试补充**

**2.7 签名密钥整理**

## 三、v1.2.0 — 导出系统

### 目标
打通从"故事设计"到"可交付作品"的导出链路。

### 功能点

**3.1 WebGAL 脚本导出**
- 将时间轴事件 + 角色对话导出为 WebGAL 兼容脚本
- 支持场景切换、角色立绘指令、BGM 指令映射
- 后端 `server/services/exporters/webgal.ts` 已有基础实现

**3.2 ZIP 完整导出/导入**
- 整个工作区打包为 ZIP（含事件/角色/伏笔/书签/地图/资产）
- 支持跨设备迁移和备份
- 后端 `server/routes/workspaces/import-export.ts` 已有基础 API

**3.3 JSON 结构化导出**
- 导出为标准化 JSON 格式
- 支持选择性导出（仅事件/仅角色/仅关系图）

### 预估工作量
- WebGAL 导出：3-5 天
- ZIP 导出/导入：2-3 天
- JSON 导出：1-2 天

## 四、v1.3.0 — 体验升级

### 目标
提升交互品质和用户效率。

### 改进方向

**4.1 全文搜索与替换**
- 后端 `/api/search` schema 已定义（`searchQuery` / `replaceBody`）
- 需实现后端路由 + 前端搜索面板
- 支持范围过滤（仅事件/仅角色/仅伏笔）

**4.2 版本历史（时光机）**
- `revisions` 表已创建，后端路由已实现
- 前端需实现版本历史面板 + diff 视图 + 回滚操作

**4.3 微交互升级**
- 时间轴拖拽动画优化（弹性回弹 + ghost 预览）
- 事件卡片 hover 信息浮层（显示关联角色/伏笔摘要）
- 面板切换过渡动画编排

**4.4 响应式优化**
- 小屏幕 StatusBar 布局修复
- 面板宽度拖拽调整（替代固定宽度）
- 时间轴缩放级别记忆（每个工作区独立保存）

### 预估工作量
- 全文搜索：3-4 天
- 版本历史：3-4 天
- 微交互：3-5 天
- 响应式：2 天

## 五、v2.0.0 — 远期方向（按需选择）

以下为远期可选方向，根据用户反馈和优先级灵活调整。

**5.1 插件系统**：允许用户安装第三方扩展（主题包/导出格式/AI 模型），需设计插件 API + 安全沙箱。

**5.2 多语言扩展**：当前仅 zh-CN / en-US，可扩展日语（视觉小说核心用户群）。

**5.3 协作模式**：基于 Git 的版本管理，支持多人协作编辑（技术复杂度高，需慎重评估）。

**5.4 移动端适配**：基于三层渲染模型，实现响应式移动端布局（已有部分 CSS 基础设施）。

**5.5 云同步**：可选的云端备份和同步服务（需要后端基础设施）。
