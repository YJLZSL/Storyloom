# 大纲视图

大纲视图以层级列表的形式组织章节、场景与事件，适合从宏观到微观梳理叙事结构。

---

## 界面布局

- **左侧大纲树**：可折叠的层级结构（卷 → 章 → 节 → 场景 → 事件）
- **中央编辑区**：选中项的详细内容编辑
- **顶部筛选栏**：状态筛选、阶段筛选、轨道筛选、搜索
- **底部版本历史**：大纲版本的保存与恢复

---

## 层级结构

Storyloom 的大纲支持多级嵌套：

```
卷一：启程
├── 第一章：告别
│   ├── 场景 1：酒馆告别
│   │   └── 事件：主角收到任务委托
│   └── 场景 2：深夜出发
│       └── 事件：遭遇第一波伏击
├── 第二章：旅途
│   ├── 场景 1：穿越森林
│   └── 场景 2：遇见向导
...
```

### 创建层级

1. 在大纲树中右键 → 「新建章节/场景/事件」
2. 或使用快捷键 `Ctrl+N`（在编辑器内）
3. 拖拽事件到目标章节/场景下，改变层级归属

### 状态标记

每个大纲节点可标记状态：

| 状态 | 含义 |
|------|------|
| ✅ 已完成 | 内容已定稿 |
| 🟡 进行中 | 正在撰写 |
| ⏳ 待处理 | 仅占位，尚未构思 |
| ❌ 已废弃 | 废弃节点，可后续清理 |

### 阶段标记（起承转合）

为每个节点分配叙事阶段：
- **起**：铺垫、引入
- **承**：发展、上升
- **转**：冲突、转折
- **合**：高潮、结局

---

## 编辑内容

### 大纲编辑器

选中任意节点后，右侧编辑器显示：
- **标题**：节点名称
- **摘要**：一句话概括
- **详细内容**：富文本描述，可包含角色、地点、时间信息
- **关联事件**：绑定到时间轴上的具体事件

### 版本管理

1. 点击「保存版本」将当前大纲保存为快照
2. 版本历史列表显示所有快照，含时间戳和描述
3. 点击「恢复」可回滚到任意历史版本
4. 支持为版本添加描述，便于识别

---

## 筛选与排序

### 筛选条件

- **状态**：仅显示已完成/进行中/待处理/已废弃
- **阶段**：仅显示起/承/转/合节点
- **轨道**：仅显示特定轨道下的事件
- **搜索**：关键词搜索标题和摘要

### 拖拽排序

- 同一层级内拖拽改变顺序
- 跨层级拖拽改变父子关系
- 拖拽到轨道头可将事件分配给该轨道

---

## 大纲与时间轴的联动

大纲视图与时间轴视图数据互通：

- 在大纲中创建的事件自动出现在时间轴上
- 在时间轴上编辑的事件信息同步到大纲
- 在大纲中点击「定位到时间轴」，自动切换视图并高亮对应事件
- 大纲节点的状态颜色与轨道上事件卡片的颜色一致

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+N` | 新建同级节点 |
| `Tab` | 将节点降级为子节点 |
| `Shift+Tab` | 将节点升级为父级节点 |
| `↑` / `↓` | 上下移动选中节点 |
| `Ctrl+↑` / `Ctrl+↓` | 调整节点顺序 |
| `Space` | 折叠/展开节点 |
| `F2` | 重命名选中节点 |
| `Delete` | 删除选中节点 |

---

## 最佳实践

1. **先搭骨架，再填血肉**：先用大纲建立卷章结构，再逐步填充场景和事件
2. **状态驱动进度**：用「已完成/进行中/待处理」标记清晰显示创作进度
3. **版本快照**：每完成一个大章节，保存一个大纲版本，便于回溯
4. **关联事件**：每个场景节点尽量关联时间轴上的具体事件，保持大纲与时间轴同步
5. **善用筛选**：在审查阶段，用「起承转合」筛选检查叙事节奏是否均衡
