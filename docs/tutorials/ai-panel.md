# AI 助手面板

Storyloom 内置 AI 写作助手，支持多种大模型，帮助你在创作中克服瓶颈、激发灵感、检查逻辑。

---

## 配置 AI

### 打开配置

1. 点击左侧 SideNav 的机器人图标（🤖）打开 AI 面板
2. 点击 AI 面板顶部的「配置」按钮
3. 在弹出的配置窗口中选择提供商和模型

### 支持的提供商

| 提供商 | 模型 | 说明 |
|--------|------|------|
| **DeepSeek** | deepseek-v4-flash (免费) | 国产大模型，中文表现优秀 |
| **Kimi (Moonshot)** | kimi-k2.6 / moonshot-v1-128k | 长文本处理能力突出 |
| **智谱 GLM** | glm-4-flash (免费) | 国内领先大模型 |
| **MiniMax** | 多模型 | 多模态能力 |
| **SiliconFlow** | 多模型 | 模型聚合平台 |
| **OpenAI** | GPT-4 / GPT-3.5 | 国际通用 |
| **自定义** | 任意 OpenAI 兼容 API | 填写 Base URL 和 API Key |

### 配置项

- **API Key**：从提供商官网获取的密钥
- **Base URL**（自定义必填）：API 的完整地址
- **自定义模型名**（自定义必填）：模型标识字符串
- **温度**：控制生成随机性（0~2，越低越保守，越高越创意）
- **最大 Token**：单次对话的最大长度限制

## 使用 AI 助手

### 对话界面

AI 面板采用聊天式界面：
- **左侧**：对话历史列表
- **右侧**：当前对话的聊天记录
- **底部**：输入框和发送按钮

### 上下文感知

AI 助手会自动携带以下上下文：
- 当前工作区的角色列表
- 世界观设定
- 时间轴上的事件摘要
- 当前选中的事件详情

这让 AI 能够给出与项目内容高度相关的建议。

### 常用提示词

| 场景 | 提示词示例 |
|------|-----------|
| 续写 | 「请帮我续写当前场景的下一部分，保持角色的说话风格」 |
| 角色分析 | 「分析角色 A 和角色 B 的关系，建议 3 个冲突点」 |
| 伏笔检查 | 「检查当前大纲中是否有未回收的伏笔」 |
| 世界观补全 | 「为主角所在的王国设计 3 条独特规则」 |
| 情节优化 | 「当前时间轴上，第 5 章和第 6 章之间缺少过渡，建议一个过渡事件」 |

## 隐私说明

- **API Key 本地存储**：你的 API Key 仅保存在本地，不会上传到 Storyloom 服务器
- **数据按需发送**：AI 对话时仅发送你明确授权的上下文数据（角色、事件等）
- **可选关闭**：不使用 AI 时，无需配置任何 API Key

---

## 最佳实践

1. **分块请求**：不要一次让 AI 生成整章内容，而是分场景、分段落请求，保持控制权
2. **利用上下文**：在对话前先切换到相关事件或角色，让 AI 基于准确上下文回答
3. **多模型对比**：对重要创作决策，用不同模型（如 DeepSeek + Kimi）分别请求，对比建议
4. **保存优质对话**：将 AI 给出的优质建议手动保存到事件描述或角色笔记中
5. **温度调节**：初稿生成用高温度（1.0~1.5），修改润色用低温度（0.3~0.7）

---

> ⚠️ 注意：AI 生成的内容仅供参考，请始终以自己的创作判断为准。AI 可能产生不符合世界观或角色设定的内容，使用时请仔细甄别。
