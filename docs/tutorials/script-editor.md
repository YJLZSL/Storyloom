# 剧本编辑器

剧本编辑器是 Storyloom 中专注于单一场景台词与演出撰写的视图，支持角色对话、旁白、演出指令的混合编排，并可一键导出到 WebGAL 视觉小说引擎。

---

## 界面布局

- **左侧角色列表**：当前场景可用的角色
- **中央编辑区**：剧本正文，支持 Markdown 格式
- **右侧预览**：实时预览剧本的渲染效果
- **底部工具栏**：导出、格式切换、字数统计

---

## 剧本格式

Storyloom 使用类 Markdown 的轻量标记语法：

### 角色对话

```
[角色名] 对话内容
```

例如：
```
[林夕] 这条路走了三年，今天终于要到尽头了。
```

### 旁白/叙述

不使用方括号，直接书写：

```
雨从凌晨开始下，到现在也没有停的意思。街道上的行人匆匆，没有人注意到站在路灯下的那个身影。
```

### 演出指令

使用尖括号标注演出指令：

```
<bg: 夜晚街道>
<music: 悲伤钢琴>
<effect: 雨滴>
```

### 分支选择

使用选择题语法：

```
[选择]
- 跟上去 → 场景_跟上去
- 转身离开 → 场景_转身离开
```

---

## 角色引用

在剧本编辑器中输入 `[` 时，自动弹出角色选择列表，快速插入角色名。列表中的角色来自当前工作区的「角色面板」。

---

## 场景管理

- **场景**：剧本的基本单元，对应时间轴上的一个事件
- **场景切换**：在剧本中标记场景切换点，如 `<scene: 场景二>`
- **场景关联**：每个场景必须关联时间轴上的一个事件，保持剧本与结构的同步

---

## 导出到 WebGAL

Storyloom 支持将剧本导出为 WebGAL 格式，用于视觉小说制作。

### 导出步骤

1. 在剧本编辑器中完成场景撰写
2. 点击「导出 → WebGAL」
3. 选择导出范围（当前场景 / 全部场景）
4. 选择输出目录
5. 生成的文件包含：
   - `index.html`：WebGAL 入口
   - `scenes/`：场景文件（按章节组织）
   - `config.txt`：引擎配置

### 导出规则

| Storyloom 语法 | WebGAL 语法 |
|---------------|-------------|
| `[角色名] 对话` | `角色名:对话` |
| `<bg: 图片名>` | `changeBg:图片名` |
| `<music: 音乐名>` | `bgm:音乐名` |
| `<effect: 效果名>` | `playEffect:效果名` |
| `[选择] - 选项 → 场景` | `choose:选项:场景` |

## 最佳实践

1. **先写结构，再填台词**：在时间轴上规划好场景，再进入剧本编辑器写台词
2. **角色语音一致性**：每个角色设定独特的说话风格（用词习惯、句式长度、口头禅），在剧本中保持一致
3. **演出指令精简**：不要每句话都加演出指令，只在关键转折处使用，给画师/演出留空间
4. **定期导出预览**：每完成一章，导出到 WebGAL 预览，检查节奏和演出效果
