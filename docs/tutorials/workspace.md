# 工作区管理

工作区是 Storyloom 中所有创作数据的容器。每个工作区独立存储事件、角色、世界观、伏笔等数据，互不影响。

---

## 创建工作区

### 从空态界面创建

1. 打开 Storyloom，若尚无工作区，会显示空态界面
2. 点击「新建工作区」或「创建第一个工作区」
3. 在弹窗中填写：
   - **名称**（必填）：工作区名称，如「夜之城传说」
   - **描述**（可选）：简短说明，如「赛博朋克背景下的侦探小说」
   - **模板**（可选）：选择预设结构
4. 点击「创建」

### 预设模板

| 模板 | 说明 |
|------|------|
| **空白** | 从零开始，自由搭建 |
| **三幕结构** | 起承转合的经典电影结构 |
| **英雄之旅** | 坎贝尔 monomyth 12 阶段 |
| **长篇小说** | 卷→章→节的层级预设 |

---

## 切换工作区

1. 返回空态界面（关闭当前工作区或重启应用）
2. 在工作区卡片列表中点击目标卡片
3. 或点击左下角 ChapterRail 中的工作区名称

---

## 导出工作区

导出可将全部数据打包为 JSON 文件，用于备份或迁移。

1. 在工作区卡片上点击「导出」按钮
2. 选择保存路径
3. 导出的 JSON 包含：工作区元信息、事件、轨道、角色、世界观、伏笔、连线、大纲版本、场景、节拍、选择支、标志、地图等全部数据

---

## 导入工作区

导入可将之前导出的 JSON 数据恢复到当前工作区。

1. 在工作区卡片上点击「导入」按钮
2. 选择之前导出的 JSON 文件
3. 选择冲突处理策略：
   - **覆盖**：用导入数据完全替换当前数据
   - **合并**：合并导入数据与现有数据（同名资源可能冲突）
   - **跳过**：跳过已存在的资源，仅导入新资源
4. 点击确认导入

> ⚠️ 注意：导入操作不可撤销，建议在导入前先导出当前工作区作为备份。

---

## 删除工作区

1. 将鼠标悬停在工作区卡片上
2. 点击右上角的删除图标（垃圾桶）
3. 确认删除提示

> ⚠️ 删除工作区会永久删除该工作区下的所有数据，包括事件、角色、世界观等。删除前请务必导出备份。

---

## 自动保存与恢复

### 自动保存

在「设置 → 偏好」中开启「自动保存」后：
- 每次编辑内容变更后，系统会自动创建快照
- 快照保存在本地数据库中
- 可通过「设置 → 关于与数据」查看自动备份路径

### 恢复数据

若意外关闭或崩溃，重启后 Storyloom 会自动检测是否有未保存的变更，并提示恢复。

---

## 数据目录

- **Windows**：`C:\Users\<用户名>\AppData\Roaming\Storyloom\data\timeline.db`
- **开发模式**：项目根目录 `data/dev.db`

可通过「设置 → 关于与数据」查看当前用户数据目录路径。

---

## 最佳实践

1. **定期导出**：每完成一个阶段性里程碑，导出一次备份
2. **命名规范**：工作区名称使用项目代号或书名，描述补充体裁和背景
3. **模板选择**：长篇小说推荐先用「长篇小说」模板建立卷章结构，再填充事件
4. **多工作区**：不同项目使用独立工作区，避免数据混乱
