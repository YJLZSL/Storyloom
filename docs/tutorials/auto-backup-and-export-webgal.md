# 自动备份与 WebGAL 导出

Storyloom 提供自动备份和 WebGAL 导出功能，保护你的创作数据，并支持一键迁移到视觉小说引擎。

---

## 自动备份

### 开启自动备份

1. 打开「设置 → 偏好」
2. 开启「自动保存」开关
3. 编辑内容变更后，系统会自动创建快照

### 备份机制

- **触发条件**：每次编辑内容变更后 5 秒（防抖）
- **存储位置**：`%APPDATA%\Storyloom\data\backups\`
- **保留策略**：保留最近 30 个自动备份，超过后自动删除最旧的
- **命名格式**：`auto-backup-<workspace-id>-<timestamp>.json`

### 手动备份

除了自动备份，你还可以随时：
- 使用「导出工作区」功能生成完整备份
- 直接复制 `%APPDATA%\Storyloom\data\` 目录到安全位置

### 恢复备份

若数据损坏或误删：
1. 关闭 Storyloom
2. 将备份文件复制到 `data/` 目录
3. 重启 Storyloom，自动检测并恢复

## WebGAL 导出

WebGAL 是一款开源视觉小说引擎，Storyloom 支持将项目直接导出为 WebGAL 格式。

### 导出范围

- **全部导出**：导出整个工作区（全部场景、角色、图片资源）
- **选择导出**：仅导出选中的场景/章节

### 导出内容

| 内容 | 说明 |
|------|------|
| 场景文件 | 每个场景对应一个 `.txt` 文件，包含对话和演出指令 |
| 角色配置 | `config.txt` 中的角色定义，包含立绘、名称、颜色 |
| 图片资源 | 场景背景、角色立绘（需提前在素材面板中配置） |
| 音频资源 | BGM、音效（需提前在素材面板中配置） |
| 树状结构 | 选择支和分支跳转自动转换为 WebGAL 的 `choose` 语法 |

### 导出后运行

1. 安装 WebGAL 引擎（或使用 WebGAL 的在线编辑器）
2. 将 Storyloom 导出的文件复制到 WebGAL 项目的 `game/` 目录
3. 运行 WebGAL 预览

### 已知限制

- 图片和音频资源需要手动准备，Storyloom 导出的是引用路径而非文件
- 复杂的演出动画（如镜头移动、角色位移）需要手动在 WebGAL 中补充
- 自定义 UI 样式需要在 WebGAL 中单独配置

---

## 最佳实践

1. **导出前检查**：使用「一致性检查」面板确认没有逻辑错误后再导出
2. **资源准备**：在 Storyloom 的素材面板中提前配置好图片和音频路径，导出时自动引用
3. **分章导出**：长篇小说建议按卷/章分多次导出，避免单文件过大
4. **版本标注**：每次导出时记录版本号，便于后续对比和回溯
