# 树状视图

树状视图将分支选择可视化为一棵决策树，便于检查多结局、剧情分叉和选择支的完整性。

---

## 界面布局

- **中央画布**：树状结构图，节点表示事件/选择支，连线表示分支路径
- **左侧导航**：快速跳转到特定分支节点
- **顶部工具栏**：缩放、筛选、搜索
- **节点详情面板**：选中节点时显示事件内容和选择支选项

---

## 树节点类型

| 节点类型 | 图标 | 说明 |
|----------|------|------|
| **根节点** | 🌳 | 故事的起点 |
| **事件节点** | ⭕ | 普通事件，无分支 |
| **选择节点** | 🔷 | 需要玩家/读者做出选择 |
| **结局节点** | ⭐ | 故事的终点，不再分叉 |
| **汇合节点** | 🔀 | 多条分支重新汇聚 |

---

## 创建分支

1. 在时间轴或大纲中，为一个事件添加「选择支」
2. 每个选择支选项成为一个新的分支路径
3. 树状视图自动根据选择支生成分支结构
4. 在分支末端继续添加事件或新的选择支

### 选择支选项

每个选择支包含：
- **选项文本**：玩家看到的选项描述
- **条件标志**（可选）：需要满足的前置条件
- **后续事件**：选择后触发的事件

---

## 连线与颜色

树状视图中的连线颜色与时间轴一致：

- **实线**：正常剧情推进
- **虚线**：闪回、伏笔、条件分支
- **条件连线**：标注条件标志，如「好感度>50」

---

## 检查完整性

树状视图的核心价值在于**检查分支完整性**：

1. **死分支检测**：标记没有结局的分支路径（死胡同）
2. **不可达节点检测**：标记没有任何路径能到达的孤立节点
3. **结局均衡**：统计各结局路径的节点数量，确保不同结局的体验均衡
4. **条件覆盖**：检查所有条件标志是否都有对应的分支处理

---

## 与视觉小说引擎的衔接

树状视图的结构可直接导出为视觉小说引擎的场景图：

- 每个节点 → 场景或选择支
- 每条连线 → 跳转或条件跳转
- 结局节点 → 游戏结束场景

导出时支持 WebGAL 格式（详见 [自动备份与 WebGAL 导出](auto-backup-and-export-webgal.md)）。

---

## 最佳实践

1. **先设计主干，再添加分支**：确保有一条完整的「真结局」路径，再添加分支
2. **分支要有意义**：避免无意义的选择，每个分支都应影响后续剧情或结局
3. **标注死分支**：暂时不打算开发的分支用「待处理」标记，避免误删
4. **定期完整性检查**：使用「一致性检查」面板发现死分支和不可达节点
