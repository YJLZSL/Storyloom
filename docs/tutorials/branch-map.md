# 分支地图

分支地图是树状视图的补充工具，以二维拓扑图的形式展示所有剧情分支和结局分布，帮助你规划多结局故事的结构。

---

## 界面布局

- **中央画布**：分支拓扑图，横向为时间轴，纵向为分支深度
- **顶部统计栏**：总节点数、分支数、结局数、死分支数
- **左侧筛选**：按结局类型、分支深度、条件标志筛选

---

## 地图元素

| 元素 | 形状 | 颜色 | 说明 |
|------|------|------|------|
| **主干** | 粗实线 | 主色 | 故事的主线，从起点到真结局 |
| **分支** | 细实线 | 次要色 | 从主干分出的剧情线 |
| **选择点** | 菱形 | 蓝色 | 需要做出选择的关键节点 |
| **结局** | 圆形 | 不同颜色 | 故事的终点 |
| **死分支** | 虚线 | 灰色 | 没有后续发展的断裂分支 |
| **条件** | 标签 | 黄色 | 进入该分支的前置条件 |

---

## 结局类型

在分支地图上，结局按类型着色：

| 结局类型 | 颜色 | 说明 |
|----------|------|------|
| **真结局** | 金色 | 最完整、最正向的结局 |
| **好结局** | 绿色 | 正面但非最理想的结局 |
| **普通结局** | 蓝色 | 中性结局 |
| **坏结局** | 红色 | 负面结局 |
| **隐藏结局** | 紫色 | 需要特殊条件触发的结局 |

---

## 平衡性分析

分支地图的核心价值在于**平衡性分析**：

### 结局均衡度

统计各结局类型所占比例，建议：
- 真结局：1 个（唯一）
- 好结局：2~3 个
- 普通结局：2~3 个
- 坏结局：1~2 个
- 隐藏结局：1~2 个

### 路径长度均衡

检查各结局路径的节点数量：
- 最短路径与最长路径的节点数差异不应超过 50%
- 否则某些结局可能过于仓促，某些又过于冗长

### 条件复杂度

- 每个选择点的选项数建议 2~4 个
- 选项超过 4 个时，玩家决策疲劳度上升
- 条件标志的使用应保持适度，过多会增加开发和测试成本

---

## 编辑操作

- **拖拽**：拖动节点改变分支结构
- **添加选择点**：右键 → 「添加选择点」
- **合并分支**：将两个分支的终点拖拽到一起，创建汇合节点
- **标记结局**：右键节点 → 「标记为结局」→ 选择结局类型
- **添加条件**：右键连线 → 「添加条件」→ 输入条件表达式

---

## 最佳实践

1. **先画地图，再写剧本**：用分支地图规划整体结构，再进入剧本编辑器填充内容
2. **标注死分支**：暂时不开发的分支标记为「待处理」，避免误删重要节点
3. **控制复杂度**：总节点数控制在 50~200 之间，过于复杂会导致玩家迷失
4. **定期均衡检查**：每新增一个结局，检查所有路径的均衡度
