# Storyloom 视觉架构 v2.0 — 织机动效系统

> 版本：v1.4.0（目标）
> 设计理念：把时间织成故事，让每一帧都有温度

---

## 一、设计哲学

### 1.1 "织机"隐喻的纵深表达

Storyloom（织叙）的核心隐喻是**织机**——时间线为经线，角色/伏笔/世界设为纬线，交织成故事的结构。当前视觉只停留在"UI 配色"层面，v2.0 要让这个隐喻贯穿**动效、光影、材质**三个维度。

### 1.2 三层渲染模型

```
┌─────────────────────────────────────────────┐
│  z-30  │  交互层 — 对话框、命令面板、悬浮按钮   │
│        │  Framer Motion（保留，负责 UI 过渡）    │
├─────────────────────────────────────────────┤
│  z-20  │  内容层 — 页面、卡片、时间轴、文本     │
│        │  React + Tailwind + GSAP（复杂动画）    │
├─────────────────────────────────────────────┤
│  z-10  │  渲染层 — 粒子、Shader、动态纹理       │
│        │  PixiJS v8（GPU 加速）                │
│        │  pointer-events: none                  │
├─────────────────────────────────────────────┤
│  z-0   │  基底 — 主题色、CSS 变量               │
│        │  Tailwind CSS 变量系统                  │
└─────────────────────────────────────────────┘
```

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **动效即叙事** | 每一个动画都服务于"织机"隐喻，而非装饰 |
| **渐进增强** | 基础功能无需 GPU，高级特效自动降级 |
| **主题可配置** | 每套主题不仅是颜色，还包括粒子、Shader、动画参数 |
| **性能优先** | 60fps 是底线，复杂场景自动降低粒子数量 |

### 2.3 双引擎策略

**保留 Framer Motion** 用于：
- 页面路由过渡（`AnimatePresence`）
- 组件挂载/卸载动画
- 悬停/点击微交互（`whileHover` / `whileTap`）
- 拖拽手势（`drag` 约束）
- 共享元素过渡（`layoutId`）

**引入 GSAP** 用于：
- 复杂时间线序列（`gsap.timeline()`）
- 滚动驱动动画（`ScrollTrigger`）
- SVG 描边/变形（`DrawSVG` + `MorphSVG`）
- 文字逐字动画（`SplitText`）
- 物理模拟（`Physics2DPlugin`）

### 3.2 PixiJS 集成模式

```tsx
// src/components/render-layer/PixiBackground.tsx
import { Application, extend } from '@pixi/react';
import { Container, Sprite, Graphics, ParticleContainer } from 'pixi.js';
import { useThemeStore } from '@/stores/useThemeStore';

extend({ Container, Sprite, Graphics, ParticleContainer });

export function PixiBackground() {
  const { currentTheme } = useThemeStore();
  const effect = currentTheme.renderLayer?.backgroundEffect;

  return (
    <Application
      width={window.innerWidth}
      height={window.innerHeight}
      backgroundAlpha={0}
      antialias={true}
      autoDensity={true}
      resolution={Math.min(window.devicePixelRatio || 1, 2)}
      eventMode="none"
      style={{ position: 'fixed', top: 0, left: 0, zIndex: 10, pointerEvents: 'none' }}
    >
      {effect === 'particles' && <ParticleSystem preset={currentTheme.renderLayer?.particlePreset} />}
      {effect === 'shader' && <ShaderEffect preset={currentTheme.renderLayer?.shaderPreset} />}
      {effect === 'texture' && <AnimatedTexture preset={currentTheme.renderLayer?.texturePreset} />}
    </Application>
  );
}
```

### 3.3 主题专属粒子效果

| 主题 | 粒子效果 | 说明 |
|------|---------|------|
| **洛生** | 暖色光尘 | 米黄色粒子缓慢飘动，模拟织机上的飞絮 |
| **子夜** | 星空 | 深蓝背景上闪烁的星星，偶尔有流星 |
| **森林** | 萤火虫 | 绿色光点在黑暗中飞舞，带有呼吸节奏 |

| **极光** | 极光波动 | 绿色/紫色光带在天空缓慢波动 |

### 4.2 主题切换动画

主题切换时，不是瞬间切换颜色，而是有一个**过渡动画**：
- 颜色渐变过渡（~500ms）
- 粒子系统淡出旧主题、淡入新主题（~800ms）
- 卡片颜色渐变（~300ms）

## 五、关键场景动画设计

### 5.1 应用启动 — 织机启动

**目标**：让用户第一次打开应用时，感受到"织机启动"的仪式感。

**动画序列**（GSAP Timeline）：
1. **0.0s** — 背景从纯黑渐变为当前主题色（~800ms）
2. **0.2s** — 织机 SVG 线条从中心向两侧绘制（DrawSVG，~1200ms）
3. **0.8s** — 经线（竖线）从上方落下，带有弹性效果（~600ms）
4. **1.2s** — 纬线（横线）从左侧滑入，与经线交织（~600ms）
5. **1.8s** — 交织点（小圆点）逐个亮起，形成织机图案（~400ms）
6. **2.0s** — 品牌文案"把时间织成故事"逐字出现（SplitText，~600ms）
7. **2.5s** — 整个启动画面淡出，主界面淡入（~500ms）

### 5.2 页面切换 — 叙事转场

**目标**：页面切换不是简单的淡入淡出，而是有叙事感的转场。

**当前方案**（Framer Motion）：
```tsx
const viewVariants = {
  initial: { opacity: 0, y: 8, scale: 0.98 },
  animate: { opacity: 1, y: 0, scale: 1 },
  exit: { opacity: 0, y: -8, scale: 0.98 },
};
```

**v2.0 方案**（GSAP Timeline）：
1. 旧页面元素按优先级逐个退出（先内容、后卡片、最后标题）
2. 过渡期间，背景粒子加速旋转（~200ms）
3. 新页面元素按优先级逐个进入（先标题、后卡片、最后内容）
4. 时间轴轨道上的事件节点以"编织"方式出现（从左到右，像织布一样）

### 5.3 时间轴滚动 — 视差滚动

**目标**：时间轴滚动时，背景、轨道、事件卡片有不同的滚动速度，产生深度感。

**方案**（GSAP ScrollTrigger）：
- 背景层（z-10）：滚动速度 0.3x（最慢，产生远景感）
- 轨道头（z-20）：滚动速度 1.0x（正常速度）
- 事件卡片（z-20）：滚动速度 1.0x，但进入视口时有入场动画
- 粒子层（z-10）：滚动速度 0.5x，带有视差偏移

### 5.4 关系图 — 发光节点

**目标**：关系图中的节点和连线有更丰富的视觉效果。

**方案**（PixiJS 渲染）：
- 节点：带有发光效果（Glow Filter），hover 时放大并增强发光
- 连线：带有脉动效果（Pulsing），粗细随关系强度变化
- 拖拽节点时：连线有弹性效果（Spring Physics）
- 选中节点时：相关节点高亮，无关节点变暗

### 5.5 Zen Mode — 沉浸式星空

**目标**：Zen Mode 是全屏沉浸式写作，背景是动态星空/粒子效果。

**方案**（PixiJS + GSAP）：
- 背景：当前主题专属的粒子效果（子夜=星空、森林=萤火虫、水墨=墨滴）
- 打字时：每输入一个字符，附近有一个粒子轻微闪烁（如萤火虫被惊动）
- 长时间未输入：粒子逐渐平静，背景变暗（节能模式）
- 退出 Zen Mode：粒子向中心收缩，然后爆发散开

## 六、后端需求

### 6.1 当前后端完全兼容

当前后端（Fastify + SQLite + Drizzle ORM）**无需修改**即可支持前端渲染层：
- 所有动画数据存储在前端（JSON 配置、本地状态）
- 后端仅需提供常规的 CRUD API

### 6.2 建议新增表（Phase 2）

```typescript
// server/db/schema.ts — 新增表
export const effectPresets = sqliteTable('effect_presets', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  category: text('category').notNull(), // 'particle', 'shader', 'texture', 'transition'
  config: text('config').notNull(),      // JSON 字符串
  thumbnail: text('thumbnail'),           // 预览图路径
  createdAt: integer('created_at', { mode: 'timestamp' }).$defaultFn(() => new Date()),
});

export const projectRenderSettings = sqliteTable('project_render_settings', {
  id: text('id').primaryKey(),
  workspaceId: text('workspace_id').notNull().references(() => workspaces.id, { onDelete: 'cascade' }),
  themeId: text('theme_id').default('luosheng'),
  backgroundEffect: text('background_effect').default('none'),
  particlePresetId: text('particle_preset_id').references(() => effectPresets.id),
  shaderPresetId: text('shader_preset_id').references(() => effectPresets.id),
  customConfig: text('custom_config'), // 用户自定义 JSON
  updatedAt: integer('updated_at', { mode: 'timestamp' }).$defaultFn(() => new Date()),
});
```

## 九、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| PixiJS 在 Tauri WebView2 中性能不如预期 | 中 | 高 | Phase 0 基准测试；提供 CSS 降级方案 |
| GSAP 包体积增加过多 | 低 | 中 | Tree Shaking 按需加载；核心仅 ~23KB gzip |
| 主题 JSON 配置过大 | 低 | 低 | 配置分片加载；默认只加载当前主题 |
| 双引擎动画冲突 | 低 | 中 | 明确分层：FM 负责 DOM，GSAP 负责 Canvas/SVG |
| 构建产物体积增加 | 中 | 中 | PixiJS 按需加载；不用的插件不打包 |
