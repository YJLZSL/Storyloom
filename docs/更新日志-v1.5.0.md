# Storyloom v1.5.0 更新日志

> **发布日期**：2026-06-21
> **GitHub Release**：https://github.com/YJLZSL/Storyloom/releases/tag/v1.5.0

---

## 概述

Storyloom v1.5.0 是一次全面重构版本，借鉴了洛笙创作（Tauri 桌面应用）的 UI 设计优点，彻底重构了前端布局和功能体系。从拥挤的三栏布局升级为简洁的顶部水平导航 + 内容全宽布局，新增了资料库、AI 助手场景化、写作三栏等核心功能。

---

## 全局布局重构

### 顶部水平导航栏
- 品牌 Logo（左）+ 导航 Tab（中）+ 操作按钮（右）
- 导航 Tab：时间轴 | 角色 | 大纲 | 细纲 | 甘特 | 树状 | 统计 | 关系 | 世界 | 写作 | 资料库 | AI助手
- 选中态：底部下划线 indicator + 背景色高亮

### 左侧边栏改造
- 仅在时间轴相关视图（timeline/outline/narrative/gantt/tree/stats/relationship）显示
- 可折叠，折叠后仅显示图标
- 移除专注工具（PomodoroTimer）和每日目标（DailyGoalWidget）——移至设置页面
- 保留：搜索事件、创作工具、实用工具、工作区概览统计

### 内容区域全宽
- 非时间轴页面（写作、资料库、AI助手）使用全宽内容区域
- 时间轴页面保留三栏布局（左栏 + 中栏画布 + 右栏关联面板）

---

## 页面布局统一

### 新增共享组件
- **PageHeader** — 统一页面标题区（标题 + 副标题 + 操作按钮）
- **SearchBar** — 统一搜索栏（搜索框 + 筛选器）
- **StatsBar** — 统一统计工具栏（统计徽章 + 视图切换 + 工具按钮 + 自动保存提示）
- **EmptyState** — 改进空状态（图标 + 标题 + 描述 + 操作按钮，支持多种尺寸和变体）

### 布局模式
所有页面遵循：
```
PageHeader → SearchBar → StatsBar → ContentArea
```

## 功能增强

### 资料库（全新功能）
- **笔记管理**：创建、编辑、删除笔记，支持标题/内容/标签/文件夹
- **文件夹树**：支持嵌套文件夹，可展开/折叠，点击筛选笔记
- **标签系统**：创建、删除标签，为笔记打标签
- **快速记录**：左栏 textarea，Ctrl+Enter 快速保存为笔记
- **搜索筛选**：按关键词搜索，按文件夹/标签筛选
- **视图切换**：网格视图 / 列表视图
- **数据库**：新增 `notes`、`note_folders`、`note_tags` 三张表
- **API**：11 个 REST 端点（笔记 CRUD + 文件夹 CRUD + 标签 CRUD）

### AI 助手改进
- **场景化功能卡片**：4 张卡片网格
  - 内容分析：分析故事结构、角色发展和情节节奏
  - 灵感启发：基于设定生成创意灵感和情节建议
  - 修改建议：对文本进行润色、扩写、改写和优化
  - 结构化拆分：将大纲拆分为章节、场景和叙事单元
- **智能对话区**：上下文显示、Tab 切换（对话/创作）、工具栏（API配置/导出/导入）
- **消息列表**：用户/AI 气泡消息，支持批量选择

### 写作三栏布局（全新页面）
- **左栏**（280px，可折叠）：
  - 章节目录、分卷管理、搜索、全书字数统计
  - 章节树形列表，点击在编辑器中打开
- **中栏**（flex-1）：
  - 工具栏：撤销/重做/排版/保存/全屏
  - 标题输入框（大号字体）
  - 富文本编辑器（使用 TTextarea，后续迭代 Markdown）
  - 底部字数/字符/目标/保存状态统计栏
- **右栏**（320px，可折叠）：
  - Tab 切换：关联 / 版本历史 / AI助手
  - 关联面板：事件/角色/世界观卡片
  - AI助手：快捷对话面板（润色/扩写/总结/续写）

## 后端变更

### 新增数据库表
```sql
notes           — 资料库笔记（id, workspace_id, folder_id, title, content, tags_json, created_at, updated_at）
note_folders    — 资料库文件夹（id, workspace_id, parent_id, name, created_at）
note_tags       — 笔记标签（id, workspace_id, name, color）
```

### 新增 API 端点
```
GET    /api/workspaces/:workspaceId/notes
POST   /api/workspaces/:workspaceId/notes
GET    /api/workspaces/:workspaceId/notes/:id
PATCH  /api/workspaces/:workspaceId/notes/:id
DELETE /api/workspaces/:workspaceId/notes/:id
GET    /api/workspaces/:workspaceId/note-folders
POST   /api/workspaces/:workspaceId/note-folders
DELETE /api/workspaces/:workspaceId/note-folders/:id
GET    /api/workspaces/:workspaceId/note-tags
POST   /api/workspaces/:workspaceId/note-tags
DELETE /api/workspaces/:workspaceId/note-tags/:id
```

### 新增前端 Hooks
- `useNotes`, `useNote`, `useCreateNote`, `useUpdateNote`, `useDeleteNote`
- `useNoteFolders`, `useCreateNoteFolder`, `useDeleteNoteFolder`
- `useNoteTags`, `useCreateNoteTag`, `useDeleteNoteTag`

## 已知限制与后续计划

1. **富文本编辑器**：当前使用 TTextarea，后续迭代为 Markdown 编辑器
2. **导入文档**：资料库暂不支持 `.md` / `.txt` 文件导入
3. **文件夹重命名**：FolderTree 中尚未支持重命名操作
4. **笔记排序**：尚未支持按更新时间/创建时间/标题排序
5. **标签颜色**：数据库已支持 `color` 字段，UI 中尚未展示彩色标签
6. **写作章节数据**：当前使用 events 作为占位数据，后续需要真正的章节数据模型
