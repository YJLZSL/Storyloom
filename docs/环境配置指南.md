# Storyloom 环境配置指南

> 面向 Windows 开发者的完整环境搭建手册。
> 最后更新：v1.4.0（美术架构升级）

---

## 1. 概述

Storyloom v1.3.0 已从 Electron 迁移到 **Tauri 2.x**，开发环境在原有 Node.js 基础上新增了 **Rust 工具链** 和 **Visual Studio Build Tools 2022** 要求。本指南提供从零开始的一步步安装说明，以及常见问题排查。

### 所需工具清单

| 工具 | 最低版本 | 用途 | 必需 |
|------|---------|------|------|
| Node.js | 20.x LTS | 前端 / 后端 / sidecar | ✅ |
| npm | 随 Node.js | 包管理 | ✅ |
| Rust | 1.77.2 | Tauri 2.x 编译 | ✅ |
| VS BuildTools 2022 | 最新版 | Windows C++ 编译工具链 | ✅（Windows） |
| Python | 3.10+ | 图标生成脚本 | ⚪ 可选 |
| Git | 任意近期版 | 源码管理 | ✅ |

## 2. Node.js 安装和配置

### 2.1 下载安装

1. 访问 [nodejs.org](https://nodejs.org/)
2. 下载 **LTS** 版本（推荐 20.x 或 22.x）
3. 运行安装程序，保持默认选项

### 2.2 验证安装

打开 PowerShell 或 Git Bash：

```bash
node --version    # 期望输出：v20.x.x 或更高
npm --version     # 期望输出：10.x.x 或更高
```

### 2.3 配置 npm（推荐）

```bash
# 设置国内镜像（可选，加速下载）
npm config set registry https://registry.npmmirror.com

# 验证配置
npm config get registry
```

### 2.4 常见问题

**问题：node 命令找不到**

- 检查是否勾选 "Add to PATH" 安装选项
- 手动添加：`C:\Program Files\nodejs\` 到系统 PATH

**问题：npm install 时 better-sqlite3 编译失败**

- 确保已安装 VS BuildTools 2022（见第 4 节）
- 或尝试：`npm install --build-from-source`

## 3. Rust 工具链安装

### 3.1 下载安装

1. 访问 [rustup.rs](https://rustup.rs/)
2. 下载 `rustup-init.exe`（64-bit）
3. 运行安装程序，选择 **Default Installation**（按回车确认）

> **提示**：Rust 安装程序会自动配置 `CARGO_HOME` 和 `RUSTUP_HOME` 环境变量，并将 `cargo` / `rustc` 添加到 PATH。

### 3.2 验证安装

**关闭并重新打开** PowerShell / Git Bash（使环境变量生效），然后执行：

```bash
rustc --version     # 期望输出：rustc 1.77.2 或更高
cargo --version     # 期望输出：cargo 1.77.2 或更高
rustup --version    # 期望输出：rustup 1.x.x
```

### 3.3 更新 Rust（如需要）

```bash
rustup update
```

### 3.4 常见问题

**问题：rustc 命令找不到**

Rust 安装程序默认将工具链放入用户目录：

```
C:\Users\<用户名>\.cargo\bin
```

检查该路径是否在 PATH 中：

```powershell
$env:PATH -split ";" | Select-String "cargo"
```

如未添加，手动添加 `C:\Users\<用户名>\.cargo\bin` 到系统 PATH。

**问题：编译时出现 `linker "link.exe" not found`**

- 说明 VS BuildTools 2022 未安装或未正确配置环境变量
- 见第 4 节安装 VS BuildTools，并确保包含 C++ 工作负载

## 4. Visual Studio Build Tools 2022 安装

### 4.1 为什么需要

Tauri 2.x 在 Windows 上编译 Rust 代码需要 MSVC（Microsoft Visual C++）工具链。这是 **硬性要求**，无法跳过。

### 4.2 下载安装

1. 访问 [Visual Studio 下载页](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022)
2. 下载 **Build Tools for Visual Studio 2022**
3. 运行安装程序，选择工作负载：

   **必须勾选：**
   - ☑️ **Desktop development with C++**（"使用 C++ 的桌面开发"）

   **该工作负载包含：**
   - MSVC v143 - VS 2022 C++ x64/x86 生成工具
   - Windows 11 SDK（或 Windows 10 SDK）
   - C++ CMake tools for Windows

4. 点击 **Install**，等待下载和安装（约 3-8 GB，视网络速度 10-30 分钟）

### 4.3 验证安装

安装完成后，检查 `link.exe` 是否存在：

```powershell
Get-Command link.exe
```

期望输出类似：

```
CommandType     Name                                               Version    Source
-----------     ----                                               -------    ------
Application     link.exe                                           ...        C:\Program Files\Microsoft Visual Studio\2022\BuildTools\VC\Tools\MSVC\...\bin\Hostx64\x64\link.exe
```

### 4.4 常见问题

**问题：已安装 VS BuildTools 但 Rust 找不到链接器**

情况 A：安装时未选 "Desktop development with C++"
- 重新运行 VS BuildTools 安装程序，修改安装，勾选该工作负载

情况 B：环境变量未正确加载
- 在 **新的** PowerShell / VS Developer Command Prompt 中重试
- 或使用：`& "C:\Program Files\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvars64.bat"`

情况 C：多版本 Visual Studio 冲突
- 检查 `LIB` / `INCLUDE` 环境变量是否指向正确的 VS 版本

## 5. 环境变量配置

正常情况下，各安装程序会自动配置 PATH。若遇到编译问题，请检查以下环境变量：

### 5.1 必需 PATH 条目

```
C:\Program Files\nodejs\                          # Node.js
C:\Users\<用户名>\.cargo\bin                       # Rust 工具链
C:\Program Files\Git\bin                          # Git（如未自动添加）
```

### 5.2 VS BuildTools 环境变量（自动配置）

VS BuildTools 安装后会在系统中注册以下变量（通过 `vcvars64.bat` 或注册表）：

```
LIB     → C:\Program Files\Microsoft Visual Studio\2022\BuildTools\VC\Tools\MSVC\...\lib
INCLUDE → C:\Program Files\Microsoft Visual Studio\2022\BuildTools\VC\Tools\MSVC\...\include
```

**不要手动修改这些变量**，如有问题请重新运行 VS BuildTools 安装程序修复。

### 5.3 检查环境变量脚本

将以下内容保存为 `check-env.ps1`，运行即可快速检查：

```powershell
# check-env.ps1 - Storyloom 环境检查脚本
Write-Host "=== Storyloom 环境检查 ===" -ForegroundColor Cyan

$checks = @(
    @{ Name = "Node.js"; Command = "node --version"; Pattern = "v\d+" },
    @{ Name = "npm"; Command = "npm --version"; Pattern = "\d+" },
    @{ Name = "Rust (rustc)"; Command = "rustc --version"; Pattern = "rustc" },
    @{ Name = "Cargo"; Command = "cargo --version"; Pattern = "cargo" },
    @{ Name = "Git"; Command = "git --version"; Pattern = "git" }
)

$allOk = $true
foreach ($check in $checks) {
    try {
        $output = Invoke-Expression $check.Command 2>$null
        if ($output -match $check.Pattern) {
            Write-Host "✅ $($check.Name): $output" -ForegroundColor Green
        } else {
            Write-Host "⚠️  $($check.Name): 版本格式异常" -ForegroundColor Yellow
            $allOk = $false
        }
    } catch {
        Write-Host "❌ $($check.Name): 未安装或不在 PATH 中" -ForegroundColor Red
        $allOk = $false
    }
}

# 检查 VS BuildTools
Write-Host ""
Write-Host "检查 Visual Studio BuildTools..." -ForegroundColor Cyan
try {
    $linkPath = Get-Command link.exe -ErrorAction Stop
    Write-Host "✅ VS BuildTools link.exe: $($linkPath.Source)" -ForegroundColor Green
} catch {
    Write-Host "❌ VS BuildTools link.exe: 未找到" -ForegroundColor Red
    Write-Host "   请安装 VS BuildTools 2022 并勾选 'Desktop development with C++'" -ForegroundColor Yellow
    $allOk = $false
}

Write-Host ""
if ($allOk) {
    Write-Host "🎉 所有环境检查通过！可以开始 Storyloom 开发了。" -ForegroundColor Green
} else {
    Write-Host "⚠️ 部分环境未就绪，请根据上方提示安装缺失组件。" -ForegroundColor Yellow
}
```

运行方式：

```powershell
.\check-env.ps1
```

## 7. 常见问题排查

### Q1: `npm install` 时 `better-sqlite3` 编译失败

**原因**：缺少 Python 或 C++ 编译工具。

**解决**：

```bash
# 确保 Python 在 PATH 中
python --version

# 确保 VS BuildTools 已安装（含 C++ 工作负载）
# 然后重新安装
npm install --build-from-source
```

### Q2: `cargo build` 或 `npm run tauri:build` 报错 `linker not found`

**原因**：Rust 找不到 MSVC 链接器（`link.exe`）。

**解决**：

1. 确认 VS BuildTools 2022 已安装且含 C++ 工作负载
2. 在 **新的** PowerShell 窗口中重试（环境变量未刷新）
3. 使用 VS Developer Command Prompt：

   ```powershell
   # 启动 VS 开发者命令提示符后，再运行 cargo
   & "C:\Program Files\Microsoft Visual Studio\2022\BuildTools\Common7\Tools\Launch-VSDevShell.ps1"
   cargo build
   ```

### Q3: `npm run tauri:dev` 窗口白屏

**原因**：前端未启动或 sidecar 未就绪。

**解决**：

1. 检查控制台是否有 `Vite dev server` 启动成功日志
2. 检查是否有 `[dev-sidecar] Backend ready on port XXXX`
3. 检查 Rust 控制台是否有 panic 信息
4. 尝试清除缓存后重试：

   ```bash
   rm -rf node_modules/.vite
   rm -rf src-tauri/target
   npm run tauri:dev
   ```

### Q4: `pkg` 打包 sidecar 失败（`npm run build:sidecar`）

**原因**：`pkg` 工具可能无法正确识别某些 Node.js 原生模块。

**解决**：

```bash
# 确保已安装 pkg
npm install -g pkg

# 或检查 pkg 版本
npx pkg --version

# 如仍失败，检查 dist-server/server/sidecar-entry.js 是否存在
npm run build:server
ls dist-server/server/sidecar-entry.js
```

### Q5: Rust 编译极慢

**原因**：首次编译需要下载和编译大量 Rust 依赖。

**解决**：

- 使用国内镜像加速 Rust 依赖下载（在 `C:\Users\<用户名>\.cargo\config.toml`）：

  ```toml
  [source.crates-io]
  replace-with = 'rsproxy-sparse'

  [source.rsproxy-sparse]
  registry = "sparse+https://rsproxy.cn/index/"
  ```

- 使用 `cargo build --release` 仅在发版时运行；开发模式用 `npm run tauri:dev`（自动增量编译）

### Q6: WebView2 运行时缺失

**原因**：Windows 系统未安装 WebView2 运行时。

**解决**：

- Windows 10/11 通常已预装 WebView2
- 如缺失，下载 [WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)
- Tauri NSIS 安装程序会自动检测并提示安装 WebView2
