## 1. 总体评估

项目已完成 **V7 全面重构**（5 阶段 17 任务），重构聚焦"新骨架"搭建：新布局（AppShell + TopToolbar + SideNav + ContextPanel + StatusBar）、新状态管理（4 个域 store）、新组件（TimelineView / AIPanel / WorkspaceSelector / CommandPalette / RelationshipGraph）。基础设施修复（package.json 恢复、环境分离、Electron 单实例锁、后端 12 个 Bug 修复）、设计系统重写（2 主题 + shadcn/ui）、前端架构重写、新增功能（命令面板/快捷键系统/故事模板库/关系图谱）、测试验证全部通过。

但大量旧代码残留，新骨架与旧功能面板未对接，导致功能断链。当前正处于**V7 重构遗留问题修复阶段**，剩余重点为接入旧面板/视图到新骨架、清理 38 个残留旧文件、移除 animejs 依赖。

**完成度估算**: 前端架构 ~85%（新骨架完成，面板接入待完成）| 后端 ~90%（V7 修复 12 个 Bug）| 整体 ~80%

> ⚠️ **V7 重构遗留问题**：详见 [前端问题识别与修复任务](./前端问题识别与修复任务.md)，包含 P0 阻塞性问题 7 个、P1 残留旧代码 38 个文件、P2 未接入功能、P3 一致性问题、P4 依赖问题。

## 3. 已知问题

> ⚠️ 完整问题清单详见 [前端问题识别与修复任务](./前端问题识别与修复任务.md)，以下为摘要。

### 3.1 P0 — 阻塞性问题（功能无法使用）

| 问题 | 说明 |
|------|------|
| MainCanvas 仅渲染 timeline 视图 | 其他 5 个视图（outline/narrative/gantt/statistics/relationship）全是占位符（P0-1） |
| ContextPanel 大部分面板是占位符 | 除 ai 面板外，其余全是占位符（P0-2） |
| EventEditorDialog 完全未接入 | 没有任何文件 import 此组件，用户无法编辑事件详情（P0-3） |
| TopToolbar 新建/保存按钮无功能 | "新建事件"和"保存"按钮无 onClick（P0-4） |
| scrollToEvent 功能断链 | TimelineCanvas 未订阅 scrollToEventId，命令面板点击事件不滚动（P0-5） |
| undo/redo 功能完全断链 | useHistoryManager 是死代码，历史栈永远为空（P0-6） |
| handleSave 是假保存 | 只执行 toast.success，不调用实际保存 API（P0-7） |

### 3.3 P3 — 一致性问题（新旧并存）

- 双主题 Store 并存（themeStore vs useThemeStore）

- 双 PanelType 定义冲突（'worldbuilding' vs 'worldview'）
- 双焦点模式定义冲突
- 动画系统双轨（animejs 31 处 import vs framer-motion）
- SideNav PanelType 与 useUIStore PanelType 不一致

- @types/d3 位置不当（在 dependencies 中，应移到 devDependencies）
