# 功能需求规格

> 本文档描述 AI Timeline Creator V4.0 需要实现的各功能模块，仅描述**需要实现什么**，不规定具体实现方式。
>
> ⚠️ V7 重构后：shadcn/ui 已集成（new-york 风格，18 个基础组件位于 `src/components/ui/`），动画引擎改为 Framer Motion 11，设计系统简化为 2 主题 + 专注模式。
>
> 可用组件库：shadcn/ui（✅ V7 已集成）、TDesign React（`tdesign-react`）、IconPark 图标（`@icon-park/react`）
> 可用图标库：lucide-react（已集成）、@icon-park/react（2658 图标，29 分类）
>
> ⚠️ V7 重构后部分功能存在断链问题（视图/面板未接入、undo/redo 断链等），详见 `前端问题识别与修复任务.md`。

---

## 一、核心编辑功能

### 1.1 事件编辑器

- 创建/编辑/删除时间轴事件
- 事件字段：标题、摘要、详细描述、开始时间、结束时间、地点、关联人物、颜色标记、标签
- 人物关联：从角色列表中选择多个角色关联到事件
- 事件卡片预览：在时间轴上显示事件的关键信息（标题、时间、地点、人物头像）
- 批量操作：多选事件后批量修改颜色、标签、轨道

> ⚠️ V7 重构后 `EventEditorDialog` 完全未接入：没有任何文件 import 此组件，双击事件卡片只显示占位符。需迁移到新 store 并接入 ContextPanel。详见 `前端问题识别与修复任务.md` P0-3。

### 1.2 撤销/重做系统

- 支持所有编辑操作的撤销和重做
- 操作历史记录栈，可回溯到任意历史状态
- 快捷键 Ctrl+Z / Ctrl+Shift+Z
- 历史记录面板：查看操作历史列表

> ⚠️ V7 重构后 undo/redo 功能断链：`useHistoryManager` 是死代码，历史栈永远为空。需重新实现并挂载到 AppShell。详见 `前端问题识别与修复任务.md` P0-6。

### 1.3 自动保存

- 编辑操作后自动防抖保存（2 秒延迟）
- 定时全量保存（120 秒间隔）
- 页面关闭前保存
- 保存状态指示器（idle / saving / saved / error）
- 崩溃恢复：检测自动保存快照，提示用户恢复

> ⚠️ V7 重构后 AutoSaveStatus 未接入 StatusBar，StatusBar 中"已保存"是硬编码。详见 `前端问题识别与修复任务.md` P3-8。

### 1.4 拖拽排序

- 事件在轨道内拖拽改变顺序
- 事件跨轨道拖拽
- 拖拽时显示占位符和插入位置指示
- 拖拽释放后弹簧物理回弹效果

## 二、时间轴视图

### 2.1 时间轴画布

- 多轨道水平时间轴，每条轨道代表一条故事线
- 时间标尺：显示时间刻度，可缩放调整精度
- 事件卡片在时间轴上的位置对应其时间
- 缩放：鼠标滚轮或双指捏合缩放时间轴
- 平移：拖拽或单指滑动平移时间轴
- 虚拟滚动：支持 1000+ 事件流畅渲染

### 2.2 轨道管理

- 创建/删除/重命名轨道
- 轨道颜色自定义
- 轨道折叠/展开
- 轨道排序拖拽

### 2.3 事件关联可视化

- 在时间轴上用连线显示事件之间的关联关系
- 关联类型：因果、闪回、伏笔、平行、对比、呼应、转折
- 不同关联类型用不同颜色和线型区分
- 点击连线查看关联详情

---

## 三、大纲视图

### 3.1 大纲列表

- 按时间顺序或轨道分组显示所有事件
- 每个事件显示：标题、时间、地点、摘要、关联人物
- 支持折叠/展开轨道分组
- 支持搜索和筛选

### 3.2 大纲编辑

- 在大纲中直接编辑事件标题和摘要
- 拖拽调整事件顺序
- 右键菜单：编辑、删除、关联、移动到轨道

## 四、统计视图

### 4.1 角色出场频率图

- 柱状图/条形图展示每个角色的出场次数
- 点击柱形跳转到该角色的相关事件
- 支持按轨道筛选

### 4.2 情节密度热力图

- 时间轴上的热力图，展示事件密集程度
- 颜色深浅代表事件密度
- 可缩放查看细节

### 4.3 伏笔回收率统计

- 饼图展示伏笔状态分布（种植/发展/回收/废弃）
- 回收率百分比
- 点击扇区跳转到对应伏笔列表

### 4.4 时间线统计

- 事件总数、角色总数、轨道数
- 时间跨度
- 平均事件间隔

## 十五、主题与专注模式

### 15.1 2 主题（V7 重构后）

V7 重构后从 12 主题简化为 2 主题：

- **素纸亮色（light）**：清爽干净、专注高效，纸张质感纯色背景，适用于日间创作
- **墨黑暗色（dark）**：沉静深邃、沉浸专注，深色纯色背景，适用于夜间创作

每个主题包含：
- 独立配色方案（CSS 变量，RGB 通道格式）
- 通过 `[data-theme="light|dark"]` 切换
- 使用 View Transitions API 实现圆形扩散切换效果（500ms）
- 不支持 View Transitions API 的浏览器降级为即时切换

### 15.2 专注模式（替代旧的视觉模式）

V7 重构后移除了 3 视觉模式（cinematic/balanced/minimal），用专注模式替代：

- **专注模式（小黑屋）**：全屏写作环境，主画布全屏化，侧栏/工具栏/状态栏淡出
- 通过 `useUIStore.focusMode` 控制（`toggleFocusMode()`）
- 进入/退出专注模式使用 Framer Motion layout 动画实现平滑过渡
- 专注模式内保留必要的写作反馈（字数统计、时间显示）
- 支持定时（15/25/45/60分钟）和定字数目标
- Esc 三次退出专注模式

> ⚠️ 旧的 12 主题（专业蓝、墨黑、纸白、深夜、森林、暖棕、赛博、作家、羊皮卷、打字机、水彩、蓝图）和 3 视觉模式（cinematic/balanced/minimal）已在 V7 重构中移除，不再支持。
