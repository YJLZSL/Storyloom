## 调查背景

项目刚完成 V7 全面重构（5 阶段 17 任务），重构聚焦"新骨架"搭建：
- 新布局：AppShell + TopToolbar + SideNav + ContextPanel + StatusBar
- 新状态管理：useWorkspaceStore / useTimelineStore / useUIStore / useThemeStore
- 新组件：TimelineView / AIPanel / WorkspaceSelector / CommandPalette / RelationshipGraph

但大量旧代码残留，新骨架与旧功能面板未对接，导致功能断链。

## P0 — 阻塞性问题（功能无法使用）

### P0-1. MainCanvas 只渲染 timeline 视图，其他 5 个视图全是占位符
- **文件**：`src/components/layout/AppShell.tsx` 第 36-49 行
- **问题**：`MainCanvas` 中只有 `viewMode === 'timeline'` 渲染 `<TimelineView />`，其他 5 个视图（outline/narrative/gantt/statistics/relationship）都只显示占位符
- **关键矛盾**：`RelationshipView` 已完整实现（使用新 store + d3），却未被 MainCanvas 渲染
- **修复方案**：
  - relationship → 直接接入 `RelationshipView`（已用新 store）
  - outline/narrative/gantt/statistics → 需迁移旧组件到新 store 后接入

### P0-2. ContextPanel 大部分面板是占位符
- **文件**：`src/components/layout/ContextPanel.tsx` 第 114-144 行
- **问题**：`getPanelContent` 中除 `ai` 面板实际渲染 `<AIPanel />` 外，其余全是占位符：
  - `properties` / `event-editor` → "事件编辑器 — Task 8 实现"
  - `characters` → "角色管理 — 后续任务实现"
  - `worldview` → "世界观 — 后续任务实现"
  - `foreshadowing` → "伏笔 — 后续任务实现"
  - `connections` → "关联 — 后续任务实现"
- **额外问题**：`consistency`、`shortcuts` 在 `useUIStore.ts` PanelType 中定义，但 ContextPanel switch 未处理
- **修复方案**：迁移旧面板组件到新 store 后接入

### P0-3. EventEditorDialog 完全未接入
- **文件**：`src/components/events/EventEditorDialog.tsx`（370+ 行）
- **问题**：没有任何文件 import 此组件。`TimelineEventCard.tsx` 双击调用 `setActivePanel('event-editor')`，但 ContextPanel 只显示占位符
- **后果**：用户无法编辑事件详情
- **额外问题**：EventEditorDialog 自身使用旧 store（useAppStore、useThemeStore）和 micro-interactions.ts，需迁移
- **修复方案**：迁移到新 store + 接入 ContextPanel

### P0-4. TopToolbar "新建事件" 和 "保存" 按钮无功能
- **文件**：`src/components/layout/TopToolbar.tsx`
- **问题**：
  - "新建事件" 按钮（Plus 图标）无 `onClick`
  - "保存" 按钮（Save 图标）无 `onClick`
- **修复方案**：新建事件打开 EventEditorDialog（新建模式）；保存触发自动保存

### P0-5. scrollToEvent 功能断链
- **文件**：`src/components/timeline/TimelineCanvas.tsx`
- **问题**：`useTimelineStore` 有 `scrollToEventId` 和 `scrollToEvent`，`CommandPalette` 调用 `scrollToEvent(event.id)`，但 `TimelineCanvas` 未订阅 `scrollToEventId`
- **后果**：命令面板点击事件搜索结果时，时间轴不滚动
- **修复方案**：TimelineCanvas 订阅 scrollToEventId，useEffect 中滚动到对应事件

### P0-6. undo/redo 功能完全断链
- **文件**：`src/components/command-palette/commands.ts`、`src/components/history/useHistoryManager.tsx`
- **问题**：
  - `handleUndo`/`handleRedo` 只调用 `useHistoryStore.getState().undo()/redo()`，但不应用返回的 HistoryRecord 到 API
  - `useHistoryManager.tsx` 是死代码，未被使用
  - 活跃代码从不调用 `pushHistoryRecord`
- **后果**：历史栈永远为空，undo/redo 永远显示"无可撤销操作"
- **修复方案**：重新实现 useHistoryManager 并挂载到 AppShell，在事件/角色等 CRUD 操作中推送历史记录

### P0-7. handleSave 是假保存
- **文件**：`src/components/command-palette/commands.ts` 第 159-161 行
- **问题**：`handleSave` 只执行 `toast.success('已保存')`，不执行实际保存
- **修复方案**：调用后端 auto-save API 保存当前工作区快照
