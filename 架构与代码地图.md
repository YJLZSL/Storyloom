### 2.2 状态管理

**V7 重构后按域拆分为 4 个 store（推荐使用）+ historyStore：**

| Store | 文件 | 职责 |
|-------|------|------|
| useWorkspaceStore | `src/stores/useWorkspaceStore.ts` | 当前工作区 ID（persist 持久化）|
| useTimelineStore | `src/stores/useTimelineStore.ts` | 视图模式(timeline/outline/narrative/gantt/statistics/relationship)、缩放、选中事件/角色、scrollToEventId、连线显示 |
| useUIStore | `src/stores/useUIStore.ts` | 活动面板(activePanel)、面板宽度(280-480)、专注模式、命令面板开关、设置开关（persist 面板宽度）|
| useThemeStore | `src/stores/useThemeStore.ts` | 2 主题(light/dark)切换 + View Transitions API 圆形扩散（persist 持久化）|
| historyStore | `src/stores/historyStore.ts` | 撤销/重做历史栈 |

**旧 store（仍存在但无活跃引用，待清理）：**
- `appStore.ts` — 旧主 store
- `panelStore.ts` — 旧面板 store
- `themeStore.ts` — 旧主题 store（13 主题 + visualMode）
- `contextMenuStore.ts` — 旧上下文菜单 store
- `pomodoroStore.ts` — 旧番茄 store

**数据流模式：** 服务端数据通过 `@tanstack/react-query` hooks 获取和缓存，客户端UI状态通过 Zustand 域 store 管理。两者互补，不重叠。

### 2.4 动画系统

**V7 重构后动画引擎为 Framer Motion 11（替代 anime.js）：**

```
src/lib/
├── animations.ts        → 旧 anime.js v4 动画工具函数（待迁移后删除）
├── cinematic-effects.ts → 旧电影级视觉效果（无引用，待删除）

└── utils.ts             → cn() 工具函数 (clsx + tailwind-merge)
```

**Framer Motion 使用模式：**
- 事件卡片：`whileHover={{ y: -2 }}` 悬浮抬起，分层投影
- 面板过渡：声明式 `motion.div` + `AnimatePresence`
- 工作区选择：Framer Motion 入场动画
- 模态框/对话框：scale + opacity 弹性过渡

**CSS transitions：** hover/focus 微交互使用 CSS transitions，统一 duration/easing 令牌（见 `src/index.css` 的 `@theme` 块：`--duration-fast: 150ms` / `--duration-normal: 250ms` / `--duration-slow: 400ms`）。

> ⚠️ **V7 重构遗留**：`package.json` 中 animejs 依赖仍存在，部分旧组件仍 import animejs（31 处），待迁移到 Framer Motion 后移除。

## 7. 构建与开发

```bash
# 开发（前端 5173 + 后端 3001 同时启动）
npm run dev

# 仅前端
npm run dev:web

# 仅后端（tsx watch 热重载）
npm run dev:server

# Electron 开发模式（加载 localhost:5173）
npm run dev:electron

# 构建
npm run build           # tsc -b && vite build（前端）
npm run build:server    # tsc -p tsconfig.server.json（后端 → dist-server/）
npm run build:electron  # tsc -p tsconfig.electron.json（Electron → electron-out/）

# 完整打包（前端+后端+Electron+electron-builder）
npm run dist

# 类型检查
npm run typecheck       # tsc -b --noEmit

# 数据库
npm run db:generate     # 生成迁移文件（drizzle-kit）
npm run db:migrate      # 执行迁移
npm run db:studio       # 打开 Drizzle Studio
```

**环境分离（V7 重构后）：**
- 开发环境：`./data/dev.db`，前端 5173 + 后端 3001 + Electron 加载 localhost:5173
- 生产环境：`app.getPath('userData')/timeline.db`，Electron 加载本地 dist/index.html，后端从 dist-server 加载

**Vite 配置要点：**
- 路径别名 `@` → `./src`
- base: `'./'` (Electron file:// 协议兼容)
- 手动分包: react / query / icons 各成独立 chunk
- 开发代理: `/api` → `localhost:3001`
