### 设计决策

- **设计系统简化**：从 12 主题 + 3 视觉模式（cinematic/balanced/minimal）简化为 2 主题（素纸亮色 light / 墨黑暗色 dark）+ 专注模式
- **动画引擎替换**：选定 Framer Motion 11 作为唯一动画引擎，替代 anime.js v4（旧组件迁移中）
- **组件库集成**：集成 shadcn/ui（new-york 风格）作为基础 UI 组件库，位于 `src/components/ui/`
- **状态管理重组**：4 个域 store（useWorkspaceStore / useTimelineStore / useUIStore / useThemeStore）替代旧的 appStore/panelStore/themeStore
- **工作台布局**：采用 CSS Grid 实现固定结构（顶栏 + 侧栏 + 主画布 + 可拖拽右栏 + 状态栏）
- **环境分离**：开发用 `./data/dev.db`，生产用 `userData/timeline.db`
- **CSS 变量格式**：使用 RGB 通道格式（如 `--background: 250 247 242`），配合 Tailwind 的 `rgb(var(--background))`

### 设计决策

- 时间轴采用「外层垂直滚动 + 内层水平滚动」双容器结构：垂直方向容纳多轨道，水平方向容纳时间比例缩放内容。
- 隐藏原生水平滚动条，使用自定义 `TimelineScrollbar` 组件，保证跨浏览器视觉一致性与可操作性。
- 鼠标滚轮在时间轴区域转换为水平滚动，符合主流时间轴工具（Premiere、Figma 等）的交互习惯。
- 建立明确的 z-index 层级规范：`z-layer-backdrop(0)` / `content(10)` / `sidebar(20)` / `panel(30)` / `toolbar(40)` / `dropdown(50)` / `overlay(50)` / `drag(60)` / `modal(100)`。

## 2026-06-16 — Phase 2 时间轴拖拽排序与跨轨道移动

### 代码变更

| 文件 | 变更内容 |
|------|----------|
| `src/components/timeline/TimelineCanvas.tsx` | 新增基于 Pointer Events 的拖拽状态管理；实现跨轨道拖放目标检测、日期时间计算、事件 `trackId`/`startTime` 更新；修复 `startDate`/`endDate` 在事件无时间时的 Invalid Date 问题 |
| `src/components/timeline/TimelineTrack.tsx` | 接入拖拽回调与轨道引用注册；根据 `isDropTarget` 高亮投放轨道；被拖拽卡片半透明缩小 |
| `src/components/timeline/TimelineEventCard.tsx` | `onPointerDown` 触发拖拽开始；拖拽过程中禁用 hover 动画；被拖拽时应用视觉反馈 |

### 设计决策

- 使用原生 Pointer Events + anime.js 现有动画函数，不引入 `@dnd-kit/core` 等第三方拖拽库，保持依赖精简。
- 拖拽直接更新事件的 `trackId` 与 `startTime`，通过 `useUpdateEvent` 提交后端并刷新列表。
- 日期范围计算在无有效 `startTime` 时回退到「当前时间 ±30 天」，避免 `Math.min(...[])` 产生 `Invalid Date`。
